# Решения: Этап 4 — Настройки

## Storage Layer

Настройки и сохранения (Этап 5) используют единый абстрактный слой хранилища — `storage.ts`. Прямые обращения к `localStorage` из `settings.ts` и `saves.ts` запрещены.

```
storage.ts
├── interface StorageAdapter { get, set, remove }
├── LocalStorageAdapter   ← используется в HTML-сборке
├── TauriAdapter          ← заглушка на Этап 4; делегирует в localStorage
│                           (настоящая реализация через @tauri-apps/plugin-store — Этап 8)
└── auto-detect           ← '__TAURI__' in window → TauriAdapter, иначе Local
```

Адаптер инициализируется как **синглтон** при первом обращении (`lazy init`). Движок и автор не конфигурируют его явно — среда определяется автоматически.

## Namespace в хранилище

Все ключи движка используют префикс `ifkit:`:

```
ifkit:settings   ← объект настроек
ifkit:saves      ← слоты сохранений (Этап 5)
```

## Экспорт и импорт

Экспорт и импорт работают с **единым JSON-файлом**, покрывающим весь namespace `ifkit:*`.

```json
{
  "version": 1,
  "exported": "2026-04-22T14:30:00Z",
  "data": {
    "ifkit:settings": { ... },
    "ifkit:saves":    { ... }
  }
}
```

- `version` — для будущих миграций при изменении схемы
- Экспорт: скачивает файл через браузерный API (`<a download>`)
- Импорт: читает файл через `<input type="file">`, пишет все ключи через адаптер

Назначение: перенос сохранений и настроек между HTML-версией и Tauri-приложением.

## Схема настроек v1

```ts
interface Settings {
  theme:       'light' | 'dark' | 'system'
  fontSize:    number   // множитель: 0.8–1.4, шаг 0.1
  musicVolume: number   // 0–1 (заглушка: поле хранится, аудио не подключено до Этапа 6)
  soundVolume: number   // 0–1 (заглушка: поле хранится, аудио не подключено до Этапа 6)
}
```

Движковые умолчания (используются если автор не задал своих):

```ts
const engineDefaults: Settings = {
  theme:       'system',
  fontSize:    1.0,
  musicVolume: 0.8,
  soundVolume: 1.0,
}
```

## Авторские умолчания

Автор может переопределить умолчания в `defineGame()`:

```ts
defineGame({
  settings: {
    theme:    'dark',
    fontSize: 1.1,
  },
  state:  { ... },
  scenes: { ... },
})
```

Неуказанные поля заполняются из `engineDefaults`. Итоговые `authorDefaults` используются:

1. При первом запуске (нет данных в хранилище)
2. При нажатии кнопки «Сбросить настройки»

Таким образом, «сброс» означает возврат к замыслу автора, а не к заводским значениям движка.

## Применение настроек к DOM

```
theme:    → атрибут data-theme на <html> ("light" | "dark" | убрать атрибут для "system")
fontSize: → CSS-переменная --ifk-font-size-base на :root (${value}rem)
```

CSS для тёмной темы через `data-theme` и `@media prefers-color-scheme` уже присутствует в `style.css`.

## UI — модальное окно

Настройки открываются по кнопке `#btn-settings` в `#controls` как модальное окно.

```
┌─────────────────────────────────────────────┐
│ ✕              Настройки                    │
├─────────────────────────────────────────────┤
│ Тема:    [ Светлая ] [ Тёмная ] [Системная] │
│ Шрифт:   ────●──────────────────  1.0×      │
│ Музыка:  ──────────●────────────  0.8       │
│ Звуки:   ─────────────────────●  1.0        │
├─────────────────────────────────────────────┤
│ [Сбросить]           [Экспорт] [Импорт]     │
└─────────────────────────────────────────────┘
```

- Изменения применяются **немедленно** (не нужно нажимать «Сохранить»)
- Закрытие по ✕, по клику на backdrop, по `Escape`
- Фокус-ловушка внутри модалки (доступность)
- ARIA: `role="dialog"`, `aria-modal="true"`, `aria-labelledby`

## Отклонённые варианты

| Вопрос | Отклонённый вариант | Причина |
|--------|---------------------|---------|
| UI | Отдельная страница | Разрывает контекст игры |
| UI | Выдвижная панель (drawer) | Сложнее в CSS, менее нативно |
| Storage | Отдельные ключи для каждой настройки | Сложнее экспортировать; удобнее один объект |
| Storage | Явная конфигурация адаптера в `GameConfig` | Лишний бойлерплейт для автора; авто-детект решает задачу |
| Экспорт | Раздельные файлы для настроек и сохранений | Один файл удобнее для сценария «перенести всё» |
| Сброс | Сброс к движковым defaults | Автор задумал определённые настройки — сброс должен уважать этот замысел |
