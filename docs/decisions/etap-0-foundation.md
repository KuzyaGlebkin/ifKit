# Решения: Этап 0 — Фундамент проекта

## Структура каталогов

Проект организован по принципу **сэндвича**: готовая оболочка сверху, логика движка снизу, начинка автора в середине.

```
my-game/
├── index.html          ← оболочка (подготовлена ifKit)
├── style.css           ← стили (подготовлены ifKit)
├── game.ts             ← точка входа автора
├── scenes.ts           ← сцены (вынесены в отдельный файл изначально)
├── state.ts            ← начальное состояние и тип GameState (опционально)
├── assets/
│   ├── audio/
│   └── images/
└── src/
    └── ifKit/          ← движок, разбитый по фичам
        ├── renderer.ts
        ├── scenes.ts
        ├── state.ts
        ├── saves.ts
        ├── audio.ts
        ├── i18n.ts
        ├── layout.ts
        ├── transitions.ts
        ├── snackbar.ts
        ├── settings.ts
        └── index.ts
```


## Точки входа для кастомизации

Продвинутый автор может углубляться последовательно, без барьеров:

1. `style.css` — изменение внешнего вида
2. `index.html` — изменение разметки оболочки
3. `src/ifKit/*.ts` — изменение поведения движка

Каждый файл в `src/ifKit/` отвечает за одну фичу и может быть изменён или заменён независимо.

## Паттерн game.ts

Автор вызывает `defineGame()` явно. Магии нет — всё видно в одном вызове.

```ts
// game.ts
import { defineGame } from './src/ifKit'
import { scenes } from './scenes'
import { initialState } from './state'

defineGame({
  state: initialState,
  static: (state) => {
    P(`HP: ${state.hp}`)
  },
  scenes,
})
```

```ts
// state.ts
export const initialState = {
  hp: 10,
  gold: 0,
}

export type GameState = typeof initialState
```

```ts
// scenes.ts
import type { GameState } from './state'

export const scenes = {
  start: (state: GameState) => {
    P('Ты в лесу.')
    goto('clearing')
  },
  clearing: (state: GameState) => {
    P('На поляне тихо.')
  },
}
```

`state.ts` выносится отдельно чтобы избежать циклического импорта между `game.ts` и `scenes.ts`. В документации для новичков состояние сначала показывается прямо внутри `defineGame()`, `state.ts` вводится позже.

## История состояний

История **двусторонняя** — модель браузерной навигации:

- Назад: откатывает к предыдущему состоянию, сдвигает указатель
- Вперёд: возвращает отменённый шаг, если после отмены не был сделан новый выбор
- Новый выбор после отмены: redo-ветка стирается

История хранится как **линейная лента с указателем**, а не просто стек.

## Распространение

Автор скачивает ZIP-архив шаблона. Обновления движка — вручную. Автоматических обновлений нет (v1).

## Аналоги — итоги исследования

| Аналог | Что берём | Чего избегаем |
|--------|-----------|---------------|
| RenPy | откат последних N состояний, слоты сохранений (дата + скриншот), i18n | жёсткий горизонтальный формат |
| Twine | сборка в один HTML-файл | собственный язык, граф сцен в GUI |
| ChoiceScript | переходы как форма с радиокнопками | собственный язык, GUI-редактор |
| Undum | snackbar при изменении характеристик | отсутствие документации, нет сохранений |
| InSTEAD | — | стирание текста, отладка без компилятора |
