# ifKit — Дорожная карта реализации

Каждый этап включает фазу анализа перед реализацией. Анализ фиксируется в `docs/analysis/` в виде отдельных заметок.

### Анализ

## Этап 2. Состояние игры

### Анализ
- [x] Определение разделения: глобальное состояние (объект пользователя) vs. локальное (вспомогательная функция движка)
- [x] Исследование: как лучше реализовать локальные переменные сцены — замыкания, WeakMap, пространство имён по ключу сцены
- [x] Решение о формате сериализации состояния для сохранений (JSON достаточно?)
- [x] Анализ: нужен ли реактивный прокси для автоматического отслеживания изменений (для snackbar)

Решения зафиксированы в `docs/decisions/etap-2-state.md`.

### Реализация

#### Утилита клонирования (`state.ts`)
- [x] Функция `jsonClone<T>(value: T): T` — `JSON.parse(JSON.stringify(...))` с единым контрактом

#### Локальное состояние сцены (`state.ts` + `scenes.ts`)
- [x] Хранилище `sceneLocals: Record<string, Record<string, unknown>>` в `state.ts`
- [x] Функция `getSceneLocal<T extends object>(sceneKey: string, defaults: T): T`
- [x] Добавить `local<T extends object>(defaults: T): T` в тип `SceneContext` и `createContext`
- [x] Сбрасывать локали текущей сцены при переходе в другую (`runGameLoop`)

#### Snackbar (`engine.ts` + `snackbar.ts`)
- [x] Тип `WatchConfig<S>` — `{ [K in keyof S]?: (prev: S[K], next: S[K]) => string | null }`
- [x] Добавить поле `watch?: WatchConfig<S>` в `GameConfig`
- [x] Функция `diffAndNotify` — снимок watch-ключей до колбэка, дифф после, сборка строк
- [x] Интеграция `diffAndNotify` в `act` и `goto` в `scenes.ts`
- [x] DOM-компонент snackbar: очередь уведомлений, показ с авто-скрытием по таймеру
