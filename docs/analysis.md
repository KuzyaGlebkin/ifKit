## InSTEAD

Плюсы:
- низкие системные требования
- подробная документация основных функций
- мощная система объектов, позволяющая делать сложные поинт н клик текстовые квесты

Минусы:
- плохая документация специализированных фич (легко делать стандартную игру, сложно кастомизировать поведение)
- избыточность для простых игр, не использующих поинт н клик механики
- част текста выводится в динамическую область которая при обновлении стирает предыдущий текст
- слабо развитая система сохранений (см. RenPy)
- нет этапа компиляции. Если в игре несколько ошибок, то отладка выглядит так: запустил, получил сообщение об ошибке 1, исправил ошибку 1, запустил, получил сообщение об ошибке 2, исправил ошибку 2...

## RenPy

Плюсы:
- помнит последние n состояний игры, легко откатится, если не успел прочитать текст, а случайно кликнул
- хорошая система сохранений (хранит не только номер слота, но и дату со скриншотом экрана)
- хорошая интернационализация (по нажатии одной кнопки превращает в тексте программы `"текст"` в `_("текст")` и генерирует файлы перевода [в файлах только оригинальный текст, перевод сам заполняешь])

Минусы:
- заточен на формат горизантальных визуальных новелл (функции типа изменяемого размера окна, динамической ориентации экрана реализовать сложно)
- сложно делать комплексные интерфейсы (окно обмена)

## Twine 

Плюсы:
- очень простое распространение (один файл html)
- есть сохранения
- есть отмена действий

Минусы:
- слабая интеграция со сторонними средствами разработки (разработка ведётся в собственной графической программе)
- собственные языки программирования
- при разработке используется граф сцен, если сцен много работать с ним сложно
- нет адаптивности из коробки (нельзя пройтись по тексту по tab, только по кнопкам)
- плохая производительность на больших играх

## ChoiseScript

Плюсы:
- интересное решение с переходами на сдругие сцены выполненными в виде формы (список радио кнопок) отправка данных с которой происходит по кнопке Next

Минусы:
- слабая интеграция со сторонними средствами разработки (разработка ведётся в собственной графической программе)
- собственные языки программирования
- при разработке используется граф сцен, если сцен много работать с ним сложно
- нет адаптивности из коробки (нельзя пройтись по тексту по tab, только по кнопкам)
- статы спрятаны в отдельном экране

## Undum

Плюсы:
- подсвечивает строчку показывающую характеристику, которую изменило действие игрока
- для начала создания игры просто скачая архив
- отображение прогрессбара для числовых характеристик

Минусы:
- кривые анимации прокрутки
- нет адаптивности из коробки (нельзя пройтись по тексту по tab, только по кнопкам)
- нет документации
- плохая система сохранений
- не понятно как распространять
