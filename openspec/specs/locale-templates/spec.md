## ADDED Requirements

### Requirement: Канонические файлы шаблона локалей

Репозиторий SHALL содержать под `src/locales/` пару файлов `template.game.json` и `template.ui.json`. Файл `template.game.json` SHALL служить основным каталогом ключей для игровых строк, извлекаемых из авторского кода скриптом `i18n:extract`. Файл `template.ui.json` SHALL содержать записи для всех `UiKey` в том же коде `template`, синхронизируемые скриптом синхронизации UI. Оба файла SHALL попадать в граф модулей сборки без импорта из авторского `game.ts`.

#### Scenario: Файлы присутствуют в дереве исходников

- **WHEN** разработчик открывает каталог `src/locales/`
- **THEN** в нём есть `template.game.json` и `template.ui.json` (включая начальное состояние после scaffold / первого extract)

### Requirement: Статический импорт шаблонов из ifKit

Движок SHALL импортировать `template.game.json` и `template.ui.json` из модуля под `src/ifKit/` (не из авторского кода игры), чтобы сборщик включал JSON в бандл независимо от поля `locales` в `defineGame`.

#### Scenario: Автор не импортирует template в game.ts

- **WHEN** в `game.ts` нет импортов `template.game.json` / `template.ui.json`, но игра вызывает `defineGame` с собственными `locales`
- **THEN** модули шаблона по-прежнему участвуют в сборке за счёт импорта из ifKit

### Requirement: Слияние слоёв шаблона с авторскими локалями

Перед вызовом `normalizeLocalesInput` движок SHALL для **каждого** кода языка, присутствующего в авторском `locales` (и в построенном `fullRaw` с учётом `sourceLanguage`), объединить словарь `game` и словарь `ui` так, что записи автора перекрывают одноимённые ключи из шаблона, а ключи, присутствующие только в шаблоне, остаются доступны для `t` и `u` на этом языке. Псевдокод языка `template` SHALL NOT появляться в списке языков настроек.

#### Scenario: Один язык без фиктивного второго

- **WHEN** автор задаёт только `sourceLanguage: 'ru'` и `locales: { ru: { game: ruGame, ui: ruUi } }`, а в шаблоне перечислены все ключи из сцен
- **THEN** переключатель языка в настройках не добавляет отдельный язык «template» и не требует пустого `en` только ради каталога ключей

#### Scenario: Перекрытие автором

- **WHEN** для ключа `K` в `template.game.json` задано непустое значение, а в авторском `ru.game` задано другое значение для `K`
- **THEN** при активном `'ru'` функция `t` использует значение из авторской локали

### Requirement: Согласованность с извлечением ключей

Ключи в `template.game.json` SHALL обновляться только скриптом извлечения из авторского TypeScript (исключая `src/ifKit/`), как описано в `dev-tooling`. Движок SHALL не перезаписывать эти файлы в рантайме.

#### Scenario: Extract обновляет шаблон

- **WHEN** разработчик добавляет новый вызов `` t`Новая фраза` `` в сцене и запускает `npm run i18n:extract`
- **THEN** в `template.game.json` появляется ключ для этой фразы с сохранением уже введённых переводов в файле
