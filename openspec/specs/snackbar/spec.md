## ADDED Requirements

### Requirement: WatchConfig тип
`GameConfig` SHALL принимать необязательное поле `watch` типа `WatchConfig<S>`, где `WatchConfig<S>` — `{ [K in keyof S]?: (prev: S[K], next: S[K]) => string | null }`. Каждый ключ — отслеживаемая характеристика, значение — форматирующая функция.

#### Scenario: Объявление watch
- **WHEN** `defineGame` вызывается с `watch: { hp: (p, n) => \`HP: ${p} → ${n}\` }`
- **THEN** движок принимает конфигурацию без ошибок типов

#### Scenario: watch не обязателен
- **WHEN** `defineGame` вызывается без поля `watch`
- **THEN** snackbar не показывается никогда, движок работает штатно

### Requirement: Дифф watch-ключей вокруг колбэка
Движок SHALL снимать значения отслеживаемых ключей непосредственно до выполнения колбэка `act` или `goto`, и сравнивать их с актуальными значениями непосредственно после. Сравнение SHALL использовать строгое равенство (`===`). Вложенные объекты в `watch` не поддерживаются в v1.

#### Scenario: Изменение отслеживаемого ключа
- **WHEN** колбэк `act` изменяет `state.hp` с 10 до 7
- **THEN** форматирующая функция `watch.hp(10, 7)` вызывается один раз

#### Scenario: Ключ не изменился
- **WHEN** колбэк `act` не меняет `state.hp`
- **THEN** форматирующая функция `watch.hp` не вызывается

#### Scenario: Форматирующая функция вернула null
- **WHEN** `watch.gold` возвращает `null`
- **THEN** уведомление для `gold` не добавляется в очередь

### Requirement: DOM-компонент snackbar
Движок SHALL создавать DOM-элемент для отображения уведомлений. Уведомления SHALL показываться последовательно или стекаться. Каждое уведомление SHALL исчезать автоматически через фиксированное время.

#### Scenario: Показ уведомления
- **WHEN** форматирующая функция возвращает непустую строку
- **THEN** в DOM появляется элемент с этим текстом

#### Scenario: Авто-скрытие
- **WHEN** уведомление появилось
- **THEN** оно исчезает автоматически (не требует действий игрока)

#### Scenario: Несколько изменений за один act
- **WHEN** один `act` изменяет несколько отслеживаемых ключей
- **THEN** все соответствующие уведомления показываются
