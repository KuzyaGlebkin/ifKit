## ADDED Requirements

### Requirement: Favicon на основе if_fav со светлым глифом на тёмном круге

Шелл игры SHALL использовать **единый production-ассет** `src/ifk-favicon.svg` как источник правды и для пиктограммы вкладки, и для нативных иконок Tauri: геометрия и заливки SHALL соответствовать этому файлу (производный от мастера `docs/if_fav.svg`, **светлый** силуэт if с заливкой **`#f5f5f5`**, на **тёмном круге** **`#242424`**, прозрачность вне круга). Холст favicon SHALL быть **квадратным** (одинаковые ширина и высота `viewBox` / артборда), иначе команда `tauri icon` искажает растровый вывод. В `src/index.html` favicon SHALL подключаться через `<link rel="icon" type="image/svg+xml" href="ifk-favicon.svg">` (или эквивалентный путь, обрабатываемый Vite при сборке). Регенерация `src-tauri/icons/*` SHALL выполняться скриптом `npm run icons:tauri` (см. `package.json`, `tauri-integration`).

#### Scenario: Вкладка в dev и web-dist

- **WHEN** пользователь открывает приложение через `npm run dev` или открывает собранный в `dist/` single-file HTML (`<name>_<version>.html` из `npm run build`) в браузере
- **THEN** во вкладке отображается favicon со светлым символом if на тёмном круге (как в production-ассете `src/ifk-favicon.svg`)

#### Scenario: Tauri webview

- **WHEN** пользователь запускает приложение из `dist-tauri/` (или через `tauri:dev` / собранный бинарник) и смотрит заголовок окна/вкладки webview
- **THEN** отображается тот же брендовый favicon

### Requirement: Логотип-слово в панели навигации в акцентном цвете

Внутри `#controls` SHALL присутствовать логотип, геометрически соответствующий `docs/logo.svg`, с заливкой, привязанной к **`var(--ifk-color-accent)`** (например через `currentColor` на корне SVG и `color` на обёртке). Интерактивный бренд и четыре глобальные кнопки панели (`#btn-undo`, `#btn-redo`, `#btn-saves`, `#btn-settings`) SHALL находиться в **одном** нативном списке `<ul class="ifk-controls-actions">`: первый дочерний `<li>` SHALL содержать только интерактивный логотип (ссылка, см. требование «Интерактивный логотип»), каждый из следующих четырёх `<li>` SHALL содержать ровно одну из указанных кнопок с неизменными `id`. Визуально при типичной ширине окна логотип SHALL оставаться у **левого** края полосы навигации, выровненным по вертикали с кнопками, а группа из четырёх кнопок SHALL быть **прижата к правому краю** полосы (например через flex-раскладку списка и отступ у первого `li`), сохраняя порядок кнопок **слева направо**: undo → redo → saves → settings. При смене темы или пресета акцента (см. `user-settings` / `ui-theming`) цвет логотипа SHALL обновляться без перезагрузки страницы.

#### Scenario: Акцент из темы

- **WHEN** применяется светлая или тёмная тема с дефолтным акцентом из `style.css`
- **THEN** логотип отображается цветом текущего `--ifk-color-accent`

#### Scenario: Пресет акцента игрока

- **WHEN** игрок выбирает не-`default` пресет акцента в настройках
- **THEN** логотип перекрашивается в соответствующий акцент вместе с остальным UI

#### Scenario: Широкое окно — бренд слева, кнопки у правого края

- **WHEN** ширина viewport достаточна, чтобы логотип и группа кнопок помещались в одну строку в `#controls`
- **THEN** логотип остаётся у левого края панели, а группа из четырёх кнопок визуально выровнена к правому краю полосы навигации

### Requirement: Доступность и фокус

Логотип в `#controls` SHALL NOT быть ловушкой фокуса: интерактивная обёртка SHALL иметь текстовую/скрытую подпись для AT (`aria-label` или эквивалент на `<a>`). SVG логотипа MAY помечаться `aria-hidden="true"`, если доступное имя задаётся снаружи.

#### Scenario: Tab по панели

- **WHEN** пользователь последовательно нажимает Tab, перемещаясь по фокусируемым элементам `#controls`
- **THEN** порядок фокуса: ссылка бренда (первый элемент списка), затем `#btn-undo`, `#btn-redo`, `#btn-saves`, `#btn-settings` без лишних tab-stop на `<ul>` / `<li>`

### Requirement: Интерактивный логотип для входа в стартовое меню

Помимо требований к визуальному логотипу в `#controls`, движок SHALL предоставлять интерактивный контрол как **нативную ссылку** `<a>` (не `<button>`), оборачивающую логотип, с стабильным `id` `btn-brand` (или согласованным преемником, задокументированным в коде), с `href`, при активации не выполняющим навигацию документа, нарушающую SPA (обработчик предотвращает переход по умолчанию). По активации SHALL выполняться переход приложения в стартовое меню сессии согласно спецификации `session-main-menu` (включая автосохранение и остановку музыки).

#### Scenario: Активация логотипа

- **WHEN** игрок активирует интерактивный логотип кликом или клавишей активации, предусмотренной для ссылок и не противоречащей движку
- **THEN** выполняется сценарий «Клик по логотипу в навбаре» из `session-main-menu`
