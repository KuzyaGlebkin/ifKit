## ADDED Requirements

### Requirement: Favicon на основе if_fav со светлым глифом на тёмном круге

Шелл игры SHALL использовать **единый production-ассет** `src/ifk-favicon.svg` как источник правды для пиктограммы вкладки: геометрия и заливки SHALL соответствовать этому файлу (производный от мастера `docs/if_fav.svg`, **светлый** силуэт if с заливкой **`#f5f5f5`**, на **тёмном круге** **`#242424`**, прозрачность вне круга). В `src/index.html` favicon SHALL подключаться через `<link rel="icon" type="image/svg+xml" href="ifk-favicon.svg">` (или эквивалентный путь, обрабатываемый Vite при сборке). Нативные иконки приложения (Tauri) SHALL визуально соответствовать тому же арту; квадратный мастер для CLI — `src/ifk-app-icon.svg`, согласованный с `ifk-favicon.svg`.

#### Scenario: Вкладка в dev и web-dist

- **WHEN** пользователь открывает приложение через `npm run dev` или открывает собранный `dist/index.html` в браузере
- **THEN** во вкладке отображается favicon со светлым символом if на тёмном круге (как в production-ассете `src/ifk-favicon.svg`)

#### Scenario: Tauri webview

- **WHEN** пользователь запускает приложение из `dist-tauri/` (или через `tauri:dev` / собранный бинарник) и смотрит заголовок окна/вкладки webview
- **THEN** отображается тот же брендовый favicon

### Requirement: Логотип-слово в панели навигации в акцентном цвете

Внутри `#controls` SHALL присутствовать логотип, геометрически соответствующий `docs/logo.svg`, с заливкой, привязанной к **`var(--ifk-color-accent)`** (например через `currentColor` на корне SVG и `color` на обёртке). Логотип SHALL располагаться слева от кнопок панели, визуально выровнен по вертикали с кнопками. При смене темы или пресета акцента (см. `user-settings` / `ui-theming`) цвет логотипа SHALL обновляться без перезагрузки страницы.

#### Scenario: Акцент из темы

- **WHEN** применяется светлая или тёмная тема с дефолтным акцентом из `style.css`
- **THEN** логотип отображается цветом текущего `--ifk-color-accent`

#### Scenario: Пресет акцента игрока

- **WHEN** игрок выбирает не-`default` пресет акцента в настройках
- **THEN** логотип перекрашивается в соответствующий акцент вместе с остальным UI

### Requirement: Доступность и фокус

Логотип в `#controls` SHALL NOT быть ловушкой фокуса: либо он декоративен с текстовой/скрытой подписью для AT (`aria-label` на интерактивном предке или статичном контейнере), либо вынесен так, чтобы не участвовал в последовательной навигации Tab между кнопками панели без явного UX-требования. SVG логотипа MAY помечаться `aria-hidden="true"`, если доступное имя задаётся снаружи.

#### Scenario: Tab по панели

- **WHEN** пользователь последовательно нажимает Tab, перемещаясь по `#controls`
- **THEN** порядок фокуса на кнопках панели остаётся предсказуемым (логотип не вставляет лишний фокусируемый элемент без необходимости)
