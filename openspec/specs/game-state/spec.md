## ADDED Requirements

### Requirement: jsonClone utility
Движок SHALL предоставлять внутреннюю функцию `jsonClone<T>(value: T): T`, реализованную как `JSON.parse(JSON.stringify(value))`. Все операции клонирования состояния в движке SHALL использовать только эту функцию.

#### Scenario: Клонирование плоского объекта
- **WHEN** вызывается `jsonClone({ hp: 10, gold: 0 })`
- **THEN** возвращается новый объект с идентичными значениями, не связанный с оригиналом по ссылке

#### Scenario: Клонирование вложенного объекта
- **WHEN** вызывается `jsonClone({ inventory: ['sword'] })`
- **THEN** возвращается глубокая копия — мутация `result.inventory` не затрагивает оригинал

#### Scenario: Ограничение типов
- **WHEN** в объекте присутствует `Date`
- **THEN** после клонирования значение становится строкой ISO (стандартное поведение JSON — не исключение)

### Requirement: GameState type constraint
`GameConfig.state` SHALL принимать только JSON-совместимые объекты. Это ограничение SHALL быть задокументировано в JSDoc поля `state`.

#### Scenario: Корректные типы
- **WHEN** `state` содержит примитивы, массивы примитивов и вложенные объекты
- **THEN** движок принимает конфигурацию без ошибок

#### Scenario: Документация ограничения
- **WHEN** разработчик наводит курсор на поле `state` в IDE
- **THEN** JSDoc показывает предупреждение о JSON-ограничении
