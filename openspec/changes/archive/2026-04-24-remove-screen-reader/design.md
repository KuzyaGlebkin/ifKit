## Context

В каталоге `openspec/specs/` и в коде зафиксированы требования к ARIA, Tab-порядку на сцене, «умному» фокусу после рендера и к модалкам. Реализация (`layout.ts`, `scenes.ts`, `renderer.ts`, `index.html`, `engine.ts`, модалки) опиралась на эти контракты. Пользовательское тестирование шло в браузере с некорректным поведением, поэтому снятие требований и кода снижает риск закрепить ошибочные решения до повторного этапа с нормальной проверкой.

## Goals / Non-Goals

**Goals:**

- Убрать из репозитория спеку `screen-reader` (как договор) и логику `makeSceneContentFocusable` + стили focus для блочного контента сцены.
- Убрать цепочку `focusAfterRender` и сопутствующие тестовые сценарии; не вызывать программный `focus()` после рендера сцены, если в спеках не осталось требований.
- Упростить `index.html` / `engine.ts`: снять обязательные ARIA-landmark с зон; при необходимости оставить нейтральную разметку (роли по умолчанию браузера).
- Убрать с кнопок `act`/`goto` обязательные `aria-label` в рендерере; ослабить соответствующий пункт `ui-buttons`.
- Убрать из модалок ARIA+focus-trap в коде; обновить `settings-modal` / `saves-modal` — сценарии открытия/закрытия не должны требовать манипуляций фокусом (кроме косвенного поведения браузера по умолчанию).
- Сохранить визуальные и функциональные сценарии (тема, шрифты, слоты и т.д.) без ссылок на «скринридер» и без строгих a11y-контрактов.

**Non-Goals:**

- Вводить новую модель доступности в этом изменении.
- Менять API `defineGame`, игровую логику, сохранения (кроме косвенного следствия снятия фокуса).
- Подробно обновлять `docs/roadmap.md` (только по явному запросу).

## Decisions

1. **Полностью удалить `openspec/specs/screen-reader/spec.md` после мерджа дельты** (дельта помечает требования как REMOVED; пустой каталог можно удалить в задачах, чтобы не вводить в заблуждение). Альтернатива — оставить пустой `spec.md`: отклонена, чтобы не плодить мёртвые файлы.
2. **Заменить `ui-zones` не «пустотой»**, а одним кратким требованием о наличии контейнеров с нужными `id` без ARIA, чтобы вёрстка движка оставалась предсказуемой.
3. **Фокус после рендера**: не переносить на первый `button` в acts/gotos и не на `#scene-content`, если сняты требования — просто убрать вызовы. Пользователь взаимодействует мышью/как ведёт себя ОС; Tab по странице возвращается к стандартному порядку DOM без искусственного `tabindex` на абзацах.
4. **Модалки**: удалить обработчики focus trap и атрибуты `role`/`aria-*` в DOM; оставить `Escape`, backdrop, кнопку закрытия. Сценарии с «фокус на первом элементе / возврат на кнопку» заменить на нейтральные (окно открылось/закрылось) без норм про фокус.
5. **Префиксы Act:/Go to:** удалить в коде; спека `ui-buttons` — REMOVED для этого требования, визуальные классы `btn-act` / `btn-goto` остаются.

## Risks / Trade-offs

- **[Риск] Регрессия для клавиатурных пользователей и AT** — ожидаемо; зафиксировано как BREAKING в proposal до нового этапа.
- **[Риск] Сценарии тестов/E2E, завязанные на фокус** — при появлении снять или переписать; mitigation: tasks включает проверку `npm test` / ручной смоук.
- **[Риск] Несоответствие кода дельтам** — мердж OpenSpec требует полного MODIFIED; mitigation: tasks идут строго по дельтам.

## Migration Plan

1. Слить дельты спеков, затем удалить `openspec/specs/screen-reader/` (файл или каталог).
2. Снять код в порядке: `scenes.ts` (вызовы) → `layout.ts` (экспорты) → `renderer` / `engine` / html / css / модалки.
3. Откат: восстановить ветку до мерджа; отдельно хранить старые спеки в git history.
