## Context

Пользователь возвращается на экран сессионного главного меню (клик по бренду в игре): вызывается `stopMusicPlaybackForMenu()`, намерение музыки очищается, трек затухает. При этом **состояние игры и текущий ключ сцены** остаются прежними — скрытый `#content` по-прежнему соответствует последней игровой сцене с `PlayMusic()` в теле сцены.

В модалке настроек смена языка и сброс «seen» вызывают `rerender()` → `runGameLoop(currentSceneKey)`. Цикл сцены снова выполняет `PlayMusic(...)`, выставляет намерение и в конце `resolveAudioIntent()` снова запускает фоновую музыку, хотя оболочка всё ещё в режиме главного меню.

## Goals / Non-Goals

**Goals:**

- Убрать нежелательное возобновление фоновой музыки при действиях в настройках на сессионном главном меню (смена языка, сброс просмотренного контента).
- Сохранить корректное обновление строк UI/«seen»-подсветки там, где оно требуется без полного игрового цикла с аудио.
- Не ломать поведение настроек во время обычной игры (когда меню сессии не активно).

**Non-Goals:**

- Менять общую модель `PlayMusic` / `resolveAudioIntent` для нормального геймплея.
- Пересматривать политику auto-resume `AudioContext` для первого жеста пользователя (кроме случаев, если явно потребуется для согласованности).

## Decisions

1. **Условный путь вместо полного `rerender()` при активном сессионном меню**  
   Для обработчиков настроек, которые сегодня вызывают `rerender()` только ради согласованности текста/seen с новым языком или сброшенной историей просмотра, при `document.documentElement.classList.contains('ifk-session-menu-active')` не вызывать полный `runGameLoop` (или вызывать узкую процедуру без шага `resolveAudioIntent` и без повторного выполнения тела сцены с `PlayMusic`).  
   **Альтернатива:** флаг в аудиомодуле «подавлять старт музыки, пока активно меню» — дублирует знание об оболочке в двух местах; предпочтительнее не запускать цикл сцены с аудио.

2. **Единая точка проверки**  
   Вынести в `scenes.ts` (или рядом) явную функцию вида «перерисовать игровую сцену для i18n/seen, если не на сессионном меню; иначе — лёгкое обновление без аудио» — чтобы не дублировать проверку класса в каждом обработчике настроек.

3. **Сброс seen**  
   Убедиться, что при отказе от полного `runGameLoop` на меню по-прежнему обновляется то, что нужно для seen (например точечный `markAndHighlight` по текущему DOM/ключу сцены), без повторного `PlayMusic`.

## Risks / Trade-offs

- **[Risk] Расхождение между «полным» и «облегчённым» обновлением** при дальнейших фичах, которые добавят состояние, завязанное именно на `runGameLoop`.  
  **Mitigation:** документировать, что полный цикл со сценой на сессионном меню из настроек не выполняется; новые фичи проверять на этот сценарий.

- **[Risk] Другие вызовы `rerender()` с открытым меню** (если появятся) могут воспроизвести баг.  
  **Mitigation:** централизовать условие в одной функции.

## Migration Plan

Изменения только в клиентском коде движка; миграций данных и отката деплоя не требуется. Откат — revert коммита.
