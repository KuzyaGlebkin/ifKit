## Why

В карточке занятого именного слота модалки сохранений скринридер озвучивает номер слота и превью текста сцены, но **не последовательно объявляет** строки превью act и goto (как будто «перепрыгивает» к кнопкам). Игрокам на Вспомогательных технологиях теряется контекст, зачем был добавлен `actPreview` / `gotoPreview` в данных слота.

## What Changes

- Расширить доступное представление карточки слота (`role="group"`), чтобы при непустых `actPreview` и `gotoPreview` эти строки **входили в тот же логический порядок озвучивания**, что и номер и подпись сцены (без пропусков перед кнопками).
- Задать стабильную разметку/идентификаторы для строк превью act/goto, чтобы их можно было включить в связку с именем группы (например `aria-labelledby` с перечислением id в порядке чтения: номер → сцена → act → goto, плюс при необходимости дата по согласованию с существующим поведением).
- **Клавиатура:** навигация **ArrowUp / ArrowDown** по вертикальной цепочке внутри карточки слота — все строки в `.ifk-slot-info` (в порядке DOM), затем содержимое `.ifk-slot-actions` (кнопки; в режиме inline-подтверждения — текст вопроса и «Да»/«Нет»), **циклически**; строки-не-кнопки с `tabindex="-1"`, чтобы не раздувать цепочку **Tab**; визуальное **:focus-visible** для фокуса на строках.

## Capabilities

### New Capabilities

_(нет — поведение относится к существующей модалке сохранений.)_

### Modified Capabilities

- `saves-modal`: уточнить требование к скринридеру для карточки слота — занятый именной слот с непустыми превью act/goto SHALL озвучивать их в читаемом порядке вместе с номером и `ifk-slot-label`, без пропуска этих строк при обходе карточки; добавить нормативное требование к **вертикальной навигации** ArrowUp/ArrowDown внутри карточки (строки и кнопки).

## Impact

- `src/ifKit/saves-modal.ts` — разметка слота, `aria-labelledby` / id для превью act/go; обработчики `keydown` для стрелок, `applySlotLineTabindex` / `bindSlotArrowNav` по карточкам, повторное применение tabIndex после смены разметки подтверждения.
- `src/style.css` — `:focus-visible` для фокусируемых строк в `.ifk-slot-info` и для текста подтверждения в `.ifk-slot-actions`.
- `openspec/specs/saves-modal/spec.md` — сценарии для act/go и сценарии для клавиатурной навигации вдоль карточки.
