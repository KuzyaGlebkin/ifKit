## Context

Авторские файлы (`game.ts`, `scenes.ts`, `state.ts`, `style.css`, `index.html`) находятся в корне проекта. Там же лежат `README.md`, `docs/`, `LICENSE`, конфиги (`tsconfig.json`, `vite.config.ts`, `eslint.config.js`, `package.json`). Для автора это лишний шум: ему нужна только своя рабочая зона.

Каталог `src/ifKit/` уже занят движком. Авторские файлы переедут в `src/` рядом с движком — одним уровнем выше.

## Goals / Non-Goals

**Goals:**
- Авторские файлы сгруппированы в `src/` и не перемешаны с конфигами/документацией
- Сборка и dev-режим работают без изменений в поведении
- Импорты движка из авторских файлов становятся короче (`./ifKit` вместо `./src/ifKit`)

**Non-Goals:**
- Изменение структуры `src/ifKit/` (движок не трогаем)
- Изменение поведения или публичного API движка
- Переработка системы ассетов

## Decisions

### Vite root → `src/`

Vite поддерживает опцию `root`, которая указывает, где искать `index.html` и от чего строить относительные импорты в браузере.

```ts
// vite.config.ts
export default defineConfig({
  root: 'src',
  build: {
    outDir: '../dist',   // относительно root, значит D:/project/dist
    ...
  }
})
```

**Альтернатива**: держать `index.html` в корне, а остальные файлы перенести в `src/` — но тогда `index.html` всё равно остаётся мусором в корне.

**Альтернатива**: использовать `build.rollupOptions.input` вместо `root` — добавляет сложности без выигрыша.

Решение: `root: 'src'` — минимальное изменение, полностью поддерживается Vite.

### outDir корректируется

При `root: 'src'` Vite вычисляет `outDir` относительно `src/`, поэтому нужно `outDir: '../dist'` чтобы dist по-прежнему создавался в корне проекта.

### Пути импортов в авторских файлах

После переезда в `src/` относительный путь до движка меняется:

| До | После |
|---|---|
| `import ... from './src/ifKit'` | `import ... from './ifKit'` |

Нужно пройтись по `game.ts`, `scenes.ts`, `state.ts` и обновить импорты.

### tsconfig.json

Текущий `include` скорее всего включает корень (`**/*.ts`). После переноса файлов в `src/` необходимо убедиться, что `src/` по-прежнему входит в scope компилятора. В большинстве случаев ничего менять не нужно — TypeScript автоматически находит файлы.

## Risks / Trade-offs

- **[Риск] Сломанный `outDir` при `root: 'src'`** → Явно задать `outDir: '../dist'` в vite.config.ts
- **[Риск] Забытые импорты** → Поиск `./src/ifKit` по всем файлам перед коммитом
- **[Риск] eslint/prettier не находят файлы** → Проверить `eslint.config.js` и `.prettierignore` — `src/` должна быть включена (обычно уже включена по умолчанию)
