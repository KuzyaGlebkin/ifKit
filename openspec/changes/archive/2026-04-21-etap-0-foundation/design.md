## Context

ifKit — движок интерактивной литературы. Автор скачивает ZIP-шаблон, открывает `game.ts` и пишет игру на TypeScript. Движок собирается Vite в один HTML-файл, готовый к публикации. Этот этап закладывает структуру каталогов, конфигурацию сборки и публичный API движка. Все последующие этапы (рендеринг, состояние, UI) будут добавлять модули внутрь `src/ifKit/`.

## Goals / Non-Goals

**Goals:**
- Определить финальную структуру каталогов, которая не будет меняться в последующих этапах
- Настроить сборку Vite → single-file HTML с `vite-plugin-singlefile`
- Зафиксировать публичный API `defineGame()` как единственную точку входа автора
- Подключить ESLint + Prettier для единообразия кода движка
- Создать минимальный рабочий шаблон, который компилируется и запускается

**Non-Goals:**
- Реализация игрового цикла, рендеринга, состояния — это Этапы 1–2
- UI-зоны интерфейса — Этап 3
- Тесты — позднее, когда появится логика для тестирования
- npm-пакет, CLI, автообновления движка

## Decisions

### Структура каталогов: «сэндвич»

Принят принцип «сэндвича»: оболочка сверху, логика движка снизу, начинка автора в середине.

```
my-game/
├── index.html        ← оболочка ifKit
├── style.css         ← стили ifKit
├── game.ts           ← точка входа автора
├── scenes.ts         ← сцены автора
├── state.ts          ← тип и начальное состояние (опционально)
├── assets/
│   ├── audio/
│   └── images/
└── src/
    └── ifKit/        ← движок
        ├── renderer.ts
        ├── scenes.ts
        ├── state.ts
        ├── saves.ts
        ├── audio.ts
        ├── i18n.ts
        ├── layout.ts
        ├── transitions.ts
        ├── snackbar.ts
        ├── settings.ts
        └── index.ts  ← единственный публичный экспорт движка
```

**Альтернатива**: монолитный `engine.ts`. Отклонена — модульная структура позволяет автору заменить отдельный модуль, не трогая остальные.

### Точка входа: `defineGame()`

Автор вызывает `defineGame()` явно — никакой магии, никакого автозапуска по умолчанию. Сигнатура:

```ts
defineGame({
  state: initialState,    // начальное состояние игры
  static?: (state) => void, // зона постоянных элементов
  scenes,                 // объект со сценами
})
```

**Альтернатива**: дефолтный экспорт объекта конфигурации. Отклонена — явный вызов функции понятнее новичку и лучше читается.

### Разделение файлов автора

`state.ts` выносится отдельно от `game.ts` и `scenes.ts`, чтобы избежать циклического импорта: `scenes.ts` импортирует тип `GameState`, `game.ts` импортирует и то и другое. В документации для новичков состояние сначала показывается прямо в `defineGame()`, `state.ts` вводится на следующем шаге.

### Сборка: `vite-plugin-singlefile`

Vite собирает всё в один `index.html` с инлайн JS и CSS. Это позволяет «распаковать и запустить» без сервера. Для Tauri и Steam это также предпочтительный формат.

**Риск**: большие ассеты (изображения, аудио) раздувают HTML. Митигация: документируем ограничение, в v2 рассмотрим base64-оптимизацию или внешние ассеты с Tauri.

### Линтер и форматтер

ESLint с `@typescript-eslint` + Prettier. Конфиги добавляются в корень проекта (`eslint.config.js`, `.prettierrc`). Скрипты в `package.json`: `lint`, `format`.

**Альтернатива**: Biome. Отклонена — более широкая экосистема ESLint + Prettier снижает барьер входа для участников.

## Risks / Trade-offs

- **Ассеты в single-file HTML** → большой размер файла при объёмных медиа. Митигация: документировать ограничение, рекомендовать внешние ссылки для больших ассетов.
- **Ручное обновление движка** → автор копирует `src/ifKit/` вручную при обновлении. Митигация: принято решение для v1, npm-пакет — в Этапе 9.
- **Циклические импорты** → выносить `GameState` в `state.ts` — единственный способ разорвать цикл. Митигация: зафиксировано в документации шаблона.
