## MODIFIED Requirements

### Requirement: Поле meta — название и описание для стартового меню

`defineGame()` SHALL принимать необязательное поле `meta` с объектом минимум с полем **название игры** (`title`), отображаемым в стартовом меню. Поле **описание** (`description` или эквивалент) SHALL быть опциональным текстом автора под заголовком. Строки `title` и `description` SHALL трактоваться как текст на **языке источника** (`sourceLanguage`) и служить **ключами** для таблицы игровых переводов по тем же правилам, что неинтерполируемый результат `` t`…` `` для этой строки (отображаемое значение SHALL браться из записей `*.game.json` активной локали). Автор SHALL добавлять переводы в `*.game.json` для каждого целевого языка; при отсутствии перевода SHALL применяться тот же fallback, что для `t` (исходная строка-ключ).

#### Scenario: Передано title

- **WHEN** автор вызывает `defineGame({ state, scenes, meta: { title: 'Моя история', description: '…' } })` и в локали активного языка есть переводы для ключей `'Моя история'` и `'…'`
- **THEN** стартовое меню показывает переведённые название и описание в соответствующих блоках

#### Scenario: Отсутствие meta

- **WHEN** поле `meta` не передано
- **THEN** движок SHALL использовать задокументированный fallback (например имя пакета или нейтральная строка) для заголовка без аварийного завершения

#### Scenario: Смена языка обновляет заголовок меню

- **WHEN** отображается стартовое меню с переданными `meta.title` и при необходимости `meta.description`, затем пользователь меняет язык так, что меняется активная игровая локаль
- **THEN** видимые заголовок и описание SHALL соответствовать новой локали по правилам поиска ключей игровых строк
