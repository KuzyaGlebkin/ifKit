## Why

В полноэкранном стартовом меню сессии заголовок и блок описания под названием игры остаются на языке строк из `defineGame({ meta })`, тогда как кнопки («Новая игра», «Загрузить», «Настройки») уже локализуются через `u()`. При смене языка в настройках подписи кнопок обновляются, но заголовок и описание — нет. Нужно выровнять поведение с моделью i18n движка (общий разрешённый язык для `t` и `u`).

## What Changes

- Движок SHALL разрешать текст заголовка и описания стартового меню через **игровые** локали так же, как обычный текст для `` t`…` `` (поиск по ключу — исходная строка на языке источника), и заново применять строки при смене языка (в том же обновлении chrome, что уже вызывает `refreshSessionMainMenuChrome`).
- Образцовая игра SHALL содержать переводы для строк `meta.title` и `meta.description` в `en.game.json`, чтобы при английском языке меню отображалось на английском.

### Modified Capabilities

- `session-main-menu`: явное требование, что видимый заголовок и описание в стартовом меню соответствуют активному разрешённому языку локалей игры и обновляются при его смене.
- `define-game-api`: уточнение раздела про `meta` — строки `title`/`description` трактуются как ключи на языке источника для таблицы игровых переводов (как для `` t`…` `` без интерполяции); автор добавляет пары в файлы `*.game.json` для каждого целевого языка.

## Impact

- `src/ifKit/session-main-menu.ts` — хранение исходных строк `meta`, обновление `#ifk-session-menu-title-text` и `#ifk-session-menu-description` в `refreshSessionMainMenuChrome` (или общая функция разрешения строки игры).
- `src/ifKit/engine.ts` — передача в меню исходных строк для последующего разрешения.
- `src/ifKit/i18n.ts` — при необходимости экспорт вспомогательной функции разрешения статической игровой строки по ключу (та же семантика поиска, что у `t` без подстановок).
- `src/game.ts`, `src/locales/en.game.json` — ключи и переводы для образца.
