## Why

В модалке настроек переключатель темы есть, а визуально ожидаемый аналог для акцентного цвета отсутствует; у ползунков (шрифт, музыка, звук) сейчас отображаются сырые доли, что плохо читается; у секции «Язык» сломана вёрстка, а при языке по умолчанию (`auto`, пустая строка) ни одна кнопка не выглядит выбранной. Нужно выровнять UI и согласовать с тем, что реально хранится в настройках и в спеках.

## What Changes

- **Акцентный цвет**: уточнить продуктовое решение — в коде сейчас нет поля `accent` в `Settings` и нет UI; акцент — CSS-токен `--ifk-color-accent` (см. `ui-theming`). Варианты: добавить в настройки игрока выбор из пресетов/цвета с применением через `applySettings`, либо явно оставить только авторский / CSS-override вне модалки и не добавлять переключатель.
- **Ползунки «Шрифт», «Музыка», «Звуки»**: хранимые значения остаются числовыми (как в `Settings`), отображение рядом с ползунком — в **процентах** (и для шрифта — осмысленный процент относительно базового, например 100% = 1.0×) вместо «0.8» / сырых долей.
- **Язык**: CSS/layout так, чтобы группа кнопок оставалась в одной строке при нормальной ширине диалога; **активное состояние** для «язык по умолчанию» — когда `language === ''`, визуально (и в спеках) зафиксировать, какой вариант считается выбранным: отдельная кнопка «Авто/системный» или подсветка **разрешённого** языка, вычисленного из i18n (нужна одна согласованная модель с `setLanguage`).

## Capabilities

### New Capabilities

- _(нет — дорабатываем существующие)_

### Modified Capabilities

- `settings-modal`: требования к отображению ползунков (проценты), вёрстке «Язык», индикатору выбранного языка при `language: ''`, при необходимости — секция «Акцент».
- `user-settings`: при добавлении акцентного цвета в persisted settings — схема, дефолты, применение; иначе — краткое уточнение, что акцент остаётся вне `Settings` до отдельного изменения.
- `ui-theming`: (опционально) если добавляем применение выбранного акцента из настроек — ссылка на то, что игрок меняет `--ifk-color-accent` через `applySettings`.

## Impact

- `src/ifKit/settings-modal.ts`, `src/style.css` — вёрстка, подписи, синхронизация DOM.
- `src/ifKit/settings.ts` — при добавлении акцентного: тип `Settings`, `applySettings`, `mergeDefaults`, экспорт/импорт.
- Спеки: `openspec/specs/settings-modal/`, `openspec/specs/user-settings/`, при смене теминга — `openspec/specs/ui-theming/`.
