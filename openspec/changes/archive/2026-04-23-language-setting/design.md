## Context

Движок уже поддерживает многоязычность через модуль `i18n` (`initI18n`, `setLanguage`, `getAvailableLanguages`). Поле `language: string` присутствует в типе `Settings` и `engineDefaults`. Тем не менее у пользователя не было никакого UI для смены языка — выбор происходил только автоматически через `navigator.language`. Эта задача добавляет явный переключатель языка в существующее модальное окно настроек.

## Goals / Non-Goals

**Goals:**
- Отображать секцию «Язык» в модальном окне настроек, когда доступно более одного языка.
- Применять выбранный язык немедленно, сохранять в хранилище, перерисовывать текущую сцену.
- Синхронизировать активную кнопку языка со значением из `Settings` при открытии модалки и после сброса.

**Non-Goals:**
- Перевод самих UI-лейблов модального окна (они остаются на языке по умолчанию).
- Поддержка RTL-раскладки для языков с письмом справа налево.
- Отображение полных названий языков вместо кодов (например «English» вместо «en»).

## Decisions

**Условный рендеринг секции языка**
Секция строится один раз при инициализации (`buildLangRow`). Если `languages.length < 2` — возвращается пустая строка, и строка не попадает в DOM. Альтернатива — показывать/скрывать через CSS — усложнила бы синхронизацию focus trap и не давала бы преимуществ.

**Коды языков как метки кнопок**
Используем `data-lang-value` кода языка напрямую (`'en'`, `'ru'`) как текст кнопок. Это просто и не требует реестра человекочитаемых названий. Автор игры при желании может сделать обёртку с полными именами на уровне своего UI.

**Немедленное применение + rerender**
При клике на кнопку: `setLanguage(lang)` → `commit({ language: lang })` → `rerender()`. Пользователь видит изменение мгновенно, без перезагрузки страницы.

## Risks / Trade-offs

- **Одна строка не локализована** — лейбл «Язык» жёстко задан по-русски. Риск минимален: он виден только при наличии нескольких языков, где автор, вероятно, позаботится о переводе своих текстов. Митигация: лейбл можно будет сделать настраиваемым через параметры `initSettingsModal` в будущей итерации.
- **Кнопки языков рендерятся при инициализации** — добавление языков во время работы не поддерживается. Это приемлемо: набор локалей задаётся статически через `defineGame()`.
