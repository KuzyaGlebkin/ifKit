## Why

В типичной сцене каждый абзац и заголовок задаётся как `P(t\`…\`)` или `H1(t\`…\`)`: автор обязан не забывать оборачивать текст в `t`. Это шум, дублирование формы «разметка + перевод», и ошибка «забыли `t`» даёт сырой ключ в UI на другом языке. Перенос комбинации «сначала `t`, потом тег» в слой ifKit снижает когнитивную нагрузку и держит сцену ближе к авторскому тексту.

## What Changes

- Движок предоставляет **tagged template** API для основных HTML-обёрток сцены (абзац, заголовки, inline emphasis/strong и т.д.), которое **внутри** применяет ту же логику ключа и подстановок, что и `t`, затем вставляет разметку в буфер (или возвращает HTML-строку для вложения — по выбранному в дизайне набору).
- Публичные символы экспортируются из `src/ifKit` (например через `index.ts`), чтобы в `scenes.ts` не импортировать `t` **только** ради обёртки «текст сцены».
- Образцовая игра (`src/scenes.ts` и при необходимости `src/game.ts`) переводится на новый API; прямой вызов `t` остаётся там, где он уместен (например `watch`, динамические строки, сложная склейка, подписи `act`/`goto`).
- **BREAKING:** публичный API разметки сцены **намеренно ломается**: прежние имена/сигнатуры (`P`/`H1`/`em`/`strong` и капитализированные варианты, принимавшие готовую `string` и/или сочетавшиеся с `` P(t`…`) ``) **не** сохраняются для обратной совместимости. Вместо них — единый контракт на tagged template с встроенным `t` (или эквивалент по дизайну). Внутри движка допускаются невыставленные наружу примитивы «вставить HTML-строку» для редких случаев, но **не** как старые публичные экспорты. Любой проект, опиравшийся на прежние вызовы, обязан мигрировать.

## Capabilities

### New Capabilities

- (нет отдельной capability; поведение закрепляется в дельте существующей `i18n`.)

### Modified Capabilities

- `i18n`: требования к новому публичному API «разметка + тот же механизм перевода, что у `t`»; сценарии эквивалентности ключей/плейсхолдеров с существующим `t`; **BREAKING**-отказ от публичного string-API старых тегов.

## Impact

- `src/ifKit/tag-functions.ts` (или выделенный модуль), `src/ifKit/index.ts`, импорты в авторском `src/scenes.ts` / `src/game.ts`.
- Словари `locales` в `game.ts` **не** меняют формат: ключи по-прежнему строятся из исходных фраз и `{0}`… как у `t`.
- Тесты/сборка: при наличии проверок экспортов — обновить ожидания; документация/шаблоны — зафиксировать миграцию и **BREAKING**.
