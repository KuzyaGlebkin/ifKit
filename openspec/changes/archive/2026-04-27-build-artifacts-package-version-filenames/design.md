## Context

- Web-сборка (`npm run build`) использует Vite + `vite-plugin-singlefile` и по умолчанию выводит `dist/index.html`.
- Desktop-сборка Tauri v2 читает `src-tauri/tauri.conf.json`: сейчас `productName` — «ifKit Game», задан явный `version`, бинарь по умолчанию наследует имя из Cargo — в итоге имена установщиков и `.exe` не совпадают с желаемым шаблоном `ifkit_<semver>`.
- Имя и версия npm-пакета в корне: `ifkit` / `0.1.0` (`package.json`).

## Goals / Non-Goals

**Goals:**

- Единый шаблон имён артефактов: `<package.json name>_<package.json version>.<ext>` для web-HTML и согласованные префиксы для MSI/NSIS (как в примере: `ifkit_0.1.0_x64.msi`, `ifkit_0.1.0_x64-setup.exe`).
- Одна точка правды для версии приложения, совпадающая с `package.json` (в т.ч. для Tauri).

**Non-Goals:**

- Переименование `productName` / заголовка окна для отображения пользователю (можно оставить «ifKit Game» в UI установщика, если это не мешает имени файлов).
- Смена схемы версионирования (semver остаётся как есть).

## Decisions

1. **Web: имя выходного HTML** — задать в `vite.config.ts` через `build.rollupOptions.output.entryFileNames` (и при необходимости согласовать `assetFileNames` для единственного инлайн-бандла), подставляя `name` и `version` из корневого `package.json` (чтение на этапе конфигурации Vite). Это надёжнее пост-скрипта `mv` и держит спеку «один файл в dist».
   - *Альтернатива:* плагин `closeBundle` с переименованием — отклонена как лишний шаг и расхождение с картой ассетов.

2. **Tauri: префикс имён бинаря и bundle** — выставить `mainBinaryName` в `tauri.conf.json` равным `name` из `package.json` (`ifkit`), чтобы CLI именовал исполняемый файл и производные установщики от этого префикса, а не от Cargo package name / пробелов `productName`.
   - *Альтернатива:* менять только `productName` на `ifkit` — ухудшит отображаемое имя приложения; отклонена.

3. **Версия в Tauri** — использовать встроенную поддержку пути к `package.json` в поле `version` конфигурации Tauri v2 (`"../package.json"` относительно `src-tauri/`), чтобы не дублировать semver в двух файлах.
   - *Альтернатива:* скрипт синхронизации pre-commit — отклонена как хрупкая.

4. **Документация** — обновить `docs/building.md`: путь к web-артефакту и примеры имён Windows bundle после изменений.

## Risks / Trade-offs

- **[Risk]** Хэш в именах чанков при смене настроек Rollup → **Mitigation:** для single-file остаётся один entry; проверить, что favicon/SVG по-прежнему резолвятся из HTML.
- **[Risk]** На macOS/Linux шаблоны имён отличаются от Windows → **Mitigation:** в спеках зафиксировать жёстко Windows-примеры и общее правило префикса `name_version` для всех платформ, без требования суффикса `_x64` на не-Windows.
- **[Trade-off]** Отображаемое имя продукта в списке программ может по-прежнему быть «ifKit Game», а имя файла — `ifkit_*` — сознательно для UX в меню «Пуск» vs имени файла для загрузок.

## Migration Plan

1. Внести изменения в конфиги и при необходимости `Cargo.toml` (если переименование бинаря потребует согласования с `mainBinaryName` — следовать предупреждениям `tauri build`).
2. Выполнить `npm run build` и `npm run tauri:build` локально; убедиться в именах в `dist/` и `src-tauri/target/release/bundle/`.
3. Обновить CI (если есть шаги, жёстко привязанные к `dist/index.html` или старым именам установщиков).
