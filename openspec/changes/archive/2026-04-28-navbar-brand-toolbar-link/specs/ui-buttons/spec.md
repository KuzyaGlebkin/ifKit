## MODIFIED Requirements

### Requirement: Семантический список глобальных кнопок `#controls`

Элементы панели `#controls` SHALL быть сгруппированы в один нативный список: контейнер `<ul class="ifk-controls-actions">` с **пятью** дочерними `<li>`. Первый `<li>` SHALL содержать ровно одну **ссылку** интерактивного бренда (`id` `btn-brand`, нативный `<a>`), следующие четыре `<li>` SHALL каждый содержать ровно одну из кнопок `#btn-undo`, `#btn-redo`, `#btn-saves`, `#btn-settings` с неизменными `id`. Визуально список SHALL NOT показывать маркеры (`list-style: none` и согласованные отступы). Порядок элементов списка SHALL совпадать с порядком Tab по фокусируемым контролям: бренд → undo → redo → saves → settings. Контейнер `<ul>` / `<li>` SHALL NOT быть ловушкой фокуса: в последовательной навигации Tab участвуют только ссылка бренда и четыре кнопки.

#### Scenario: Разметка списка

- **WHEN** браузер загружает `src/index.html`
- **THEN** внутри `<nav id="controls">` существует один `<ul class="ifk-controls-actions">`, содержащий пять `<li>`: в первом — один `<a id="btn-brand">` с логотипом; в каждом из остальных четырёх — ровно одна кнопка с `id` из набора `btn-undo`, `btn-redo`, `btn-saves`, `btn-settings`

#### Scenario: Tab только по интерактивным контролям

- **WHEN** пользователь последовательно нажимает Tab, перемещаясь по глобальной панели
- **THEN** фокус переходит между ссылкой бренда и четырьмя кнопками в порядке разметки без остановки на `<ul>` или `<li>` как на отдельных tab-stop

#### Scenario: Согласованность с узким экраном

- **WHEN** ширина viewport не более 600px (режим только иконок на кнопках)
- **THEN** требования «Узкий экран — только иконки на `#controls`» и «Скринридер на узком экране» по-прежнему выполняются для четырёх кнопок внутри списка; доступное имя ссылки бренда остаётся осмысленным
