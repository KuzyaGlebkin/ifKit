## Context

Сейчас в `src/style.css` для `.paragraph--unseen` задано `background-color: var(--ifk-unseen-color, rgba(255, 220, 80, 0.18))` — жёлтый тинт не связан с `--ifk-color-accent`, хотя в проекте уже используется `color-mix` для согласованности с акцентом (например сегменты настроек). Класс по-прежнему выставляет `seen-content.ts`; меняется только визуальный дефолт и спеки.

## Goals / Non-Goals

**Goals:**

- Дефолтный фон непрочитанного абзаца выразить как полупрозрачный **акцент** (`--ifk-color-accent`), чтобы подсветка совпадала с темой и пресетами акцента.
- Сохранить `--ifk-unseen-color` как опциональную кастомизацию (автор/игрок переопределяет целиком).
- Сохранить читаемость: сила тинта (~15–20% смеси с прозрачным) в духе прежнего `0.18` альфа.

**Non-Goals:**

- Менять алгоритм `markAndHighlight` или требования `seen-content` к классу.
- Вводить анимации или обводки для непрочитанного текста.
- Поднимать жёсткие WCAG-контрастные требования именно к фону абзаца (тинт декоративный; основной контраст остаётся у текста/фона сцены).

## Decisions

1. **Формула дефолта** — `color-mix(in srgb, var(--ifk-color-accent) 18%, transparent)` как второй аргумент в `var(--ifk-unseen-color, …)` на правиле `.paragraph--unseen` **или** задать одну строку-дефолт для `--ifk-unseen-color` в `:root` (эквивалентно по поведению). Предпочтительно одно место: либо только фоллбек у `var()`, либо определение `--ifk-unseen-color` в `style.css` на `:root` / тёмной ветке, чтобы при желании доразличать свет/тём без дублирования формулы. *Рекомендация к реализации:* фоллбек в `background-color` у `.paragraph--unseen` — проще сопровождать.

2. **Почему не `alpha` в `rgb(from …)`** — `color-mix` уже применяется в проекте, поддержка в целевых браузерах согласована с остальными токенами; единообразие с существующими правилами.

3. **Альтернатива** — оставить жёлтый и документировать «подкрутить акцент вручную»; отклонено: дефолт должен сразу следовать бренду.

## Risks / Trade-offs

- **[Очень светлый акцент на светлом фоне]** — тинт может быть слабее, чем у жёлтого. → *Митигация:* подобрать процент (18%) близко к прежней заметности; при необходимости сдвинуть на этапе apply после визуальной проверки.
- **[Переопределение автором]** — раньше с `--ifk-unseen-color` снимали жёлтый; теперь снимают акцентный тинт. → Поведение override не ломается.

## Migration Plan

- Изменение только CSS (и дельта-спек): откат — вернуть прежний фоллбек `rgba(255, 220, 80, 0.18)`.
- Сохранения и `localStorage` не затрагиваются.

## Open Questions

- Нет: процент смеси (18%) можно зафиксировать в реализации и при необходимости скорректировать в одной строке.
