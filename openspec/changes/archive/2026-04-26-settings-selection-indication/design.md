## Context

В `settings-modal.ts` классы `--active` уже навешиваются на сегменты темы, акцента и языка; в `style.css` активное состояние использует `background-color: var(--ifk-color-accent)` и `color: var(--ifk-color-accent-fg)`. Для «Тема» и «Язык» это работает, но для «Акцент» выбранная кнопка перекрашивается тем же токеном, который пользователь как раз меняет — визуальная связь «какой пресет выбран» становится менее предсказуемой. Пользователь просит усилить индикацию для всех трёх рядов.

## Goals / Non-Goals

**Goals:**

- Сделать выбранное значение в рядах «Тема», «Акцент», «Язык» заметным без догадок (в светлой и тёмной теме).
- Для ряда «Акцент» — развязать стиль **выбранного сегмента** от «текущего акцента пресета» (или добавить вторичную индикацию: подпись, обводка, маркер).
- Минимальный объём правок: преимущественно `style.css` и при необходимости разметка/`syncToDOM` в `settings-modal.ts`.

**Non-Goals:**

- Смена логики сохранения настроек, i18n API, `aria-hidden` для блока темы/шрифта.
- Редизайн всей модалки или других рядов (шрифт, громкость).

## Decisions

1. **Отдельные токены для «выбранный сегмент» в модалке**  
   Ввести в `ui-theming` (и `style.css`) переменные вроде `--ifk-segment-selected-bg` / `--ifk-segment-selected-fg`, задаваемые в `:root` и `[data-theme="dark"]` (и согласованно с `@media (prefers-color-scheme: dark)`), **не** равные по смыслу обязательному «живому» `--ifk-color-accent` для заливки сегмента. Сегмент-кнопки в настройках для состояния «active» используют эти токены. **Результат:** при смене пресета акцента выбранная кнопка в ряду «Акцент» остаётся читаемой; глобальный акцент по-прежнему идёт из `applySettings`.

2. **Дополнительная текстовая подсказка (опционально к токенам)**  
   Рядом с подписями «Тема» / «Акцент» / «Язык» (вторая колонка или суффикс у лейбла) выводить краткое текущее значение: человекочитаемые подписи кнопок (как на кнопках) или код языка / «Авто». Обновлять в `syncToDOM` вместе с классами. **Альтернатива отклонена:** только иконка без текста — хуже для локализации и плотности UI.

3. **Язык**  
   Логика `raw === settings.language` сохраняется; индикация усиливается теми же токенами сегмента + при принятии решения (1)+(2) — текст «Авто» / `en` и т.д. у лейбла.

## Risks / Trade-offs

- **[Risk]** Дублирование «текст на кнопке + текст у лейбла» может казаться избыточным. **→ Mitigation:** если визуально достаточно токенов (1), текстовую строку (2) можно не вводить; спека оставляет место для одного из путей — в задачах явно проверить достаточность после стилей.
- **[Risk]** Новые переменные должны не ломать авторские переопределения. **→ Mitigation:** дефолты только в движке; автор может переопределить `--ifk-segment-selected-*` в своём CSS, если нужно.

## Migration Plan

Изменения только в фронтенде движка; откат — revert коммита. Сохранённые настройки не мигрируются.
