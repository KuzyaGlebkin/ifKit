## Context

Класс `.ifk-checkbox` применяется к нативным `<input type="checkbox">` в модалке настроек (музыка, звуки, переключатель «непрочитанный»/unseen) и оформляется в `src/style.css` с `accent-color` на акцентный токен. В тёмной теме **неотмеченный** квадрат у многих движков остаётся светлым, что выбивается из `--ifk-color-bg` и `--ifk-color-text`. Цель — визуально увязать неотмеченное состояние с дизайн-токенами движка, не ломая семантику и отмеченное состояние.

## Goals / Non-Goals

**Goals:**

- Неотмеченный чекбокс: **рамка** согласована с цветом текста (`--ifk-color-text`), **внутренняя заливка** — с фоном страницы/контекста (`--ifk-color-bg` или, при необходимости для контраста внутри модалки, согласованный поверхностный токен, с явным обоснованием в CSS).
- Работает в светлой теме, в `data-theme="dark"`, и при системной теме без явного `data-theme` (те же селекторы, что и для остальной палитры ifKit).
- Отмеченное сост остаётся различимым; акцент и галочка не «теряются» на фоне.

**Non-Goals:**

- Смена разметки, `class`ов или логики настроек.
- Замена нативного чекбокса кастомным React/Vue-компонентом (достаточно CSS, если браузер позволяет).
- Redesign остальных форм-контролов.

## Decisions

1. **Стилизация через `appearance: none` + явная рамка/фон для неотмеченного состояния** (предпочтительно) *или* минимальные оверлей-правки поверх нативного вида, если в целевых браузерах (Chromium, Firefox, WebKit) один подход стабильно даёт «рамка = текст, заливка = фон».
   - *Rationale:* `accent-color` не управляет фоном неотмеченного нативного чекбокса; кросс-браузерно предсказуемый вид даёт сброс `appearance` и рисование рамки/фона токенами.
   - *Альтернатива:* только `color-scheme` на `:root` — часто **недостаточно** для полного соответствия кастомным токенам ifKit.
2. **Псевдоклассы `:checked` + при необходимости `::before`/`background` для галочки** — сохранить читаемую отметку с использованием `--ifk-color-accent` (и при необходимости `--ifk-color-accent-fg` для контраста галочки).
3. **Состояния `:focus-visible`** — оставить или добавить кольцо фокуса, согласованное с существующими focus-стилями движка (без отключения фокуса).
4. **Токены:** `border-color: var(--ifk-color-text)` (или `currentColor` при `color: var(--ifk-color-text)` на `input`), `background: var(--ifk-color-bg)`; если внутри модалки фон не совпадает с корневым `--ifk-color-bg`, зафиксировать в дизайне, какой токен считать «фоном» для чекбокса (предпочтительно один токен, чтобы не плодить исключения).

## Risks / Trade-offs

- **Кастомные чекбоксы ведут себя чуть иначе по пикселям в разных движках** → сузить тест к Chromium + Firefox (и Tauri webview) как в `docs`/`building`.
- **Сложность с `indeterminate`**, если появится → сейчас не используется; отложить.
- **Контраст галочки на акценте** → проверить отмеченное состояние на светлой/тёмной теме.

## Migration Plan

Изменения только в CSS (и спеке): деплой вместе с обычной сборкой; откат — revert коммита.
