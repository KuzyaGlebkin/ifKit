## MODIFIED Requirements

### Requirement: Шаблон проекта — игровая локаль в JSON и импорт в defineGame

Каталог ключей для игровых строк (`t` и эквиваленты), синхронизируемый с кодом автора, SHALL храниться в `src/locales/template.game.json` и обновляться скриптом `i18n:extract` из `dev-tooling`; движок SHALL подключать этот файл (и парный `template.ui.json`) через статический импорт внутри `src/ifKit`, без требования импорта в `game.ts`. Переводы **целевых** языков автор MAY хранить в отдельных `src/locales/<code>.game.json` / `*.ui.json` и импортировать в `game.ts` только для реальных кодов `<code>`. Скрипт **не** генерирует машинный перевод — только синхронизирует **ключи** и сохраняет уже введённые значения.

#### Scenario: Каталог ключей не требует импорта в game.ts

- **WHEN** разработчик открывает `game.ts` образцовой игры
- **THEN** в нём **нет** обязательных импортов `template.game.json` / `template.ui.json`, при этом ключи из сцен попадают в `template.game.json` через extract и в бандл через ifKit

#### Scenario: Реальный язык по-прежнему может импортироваться из JSON

- **WHEN** автор задаёт `locales: { ru: { game: ruGame, ui: ruUi } }` с импортами из `ru.game.json` / `ru.ui.json`
- **THEN** `defineGame` принимает конфигурацию без ошибок, а ключи из шаблона подмешиваются согласно `locale-templates`

### Requirement: `LocaleBundle` в шаблоне с импортируемыми `*.game.json` и `*.ui.json`

Когда автор для кода `<code>` использует отдельные `src/locales/<code>.game.json` и `src/locales/<code>.ui.json`, он SHALL передавать в `defineGame({ locales: { … } })` значения `game` и `ui` из импортов для **каждого** такого реального `<code>`. Пара `template.game.json` / `template.ui.json` **не** импортируется в `game.ts`: она подключается движком. Плоский формат `locales` остаётся допустимым. **Образцовая** игра ifKit SHALL демонстрировать только импорты для языков, которые автор реально переводит, без фиктивных пустых локалей ради каталога ключей.

#### Scenario: Два реальных языка — импорты только для них

- **WHEN** автор смотрит `game.ts` образцовой игры с `ru` и `en` как реальными кодами
- **THEN** импорты `*.game.json` / `*.ui.json` относятся только к этим кодам (или к подмножеству), а не к `template`

#### Scenario: Согласованность пар на код

- **WHEN** для кода `by` используется отдельный `by.game.json`
- **THEN** в рекомендуемой конфигурации передаётся и `ui` из `by.ui.json`, либо пустой слой `ui` согласован с встроенным `BUILTIN_UI` после нормализации
