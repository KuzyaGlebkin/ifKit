## Why

Сейчас строки `t` / UI-ключи попадают в граф сборки только если автор явно импортирует хотя бы один JSON из `src/locales` и передаёт его в `defineGame({ locales })`. Чтобы включить извлечение ключей и не потерять их в бандле, авторы часто добавляют «второй» язык с пустыми переводами; в настройках появляется лишний пункт языка, хотя перевода нет. Нужна опорная пара файлов `template.*.json`, подключаемых внутри движка, чтобы билд и `i18n:extract` работали без засорения `game.ts` фиктивными локалями.

## What Changes

- Добавить в репозиторий (и в шаблон игры) канонические файлы `src/locales/template.game.json` и `src/locales/template.ui.json`, наполняемые/синхронизируемые tooling’ом (пустые значения для новых ключей, без машинного перевода).
- Импортировать эти модули из кода `src/ifKit` (а не из авторского `game.ts`), чтобы Vite всегда включал их в бандл без действий автора.
- При инициализации i18n сливать слои из шаблона с авторскими `locales` по согласованным правилам (ключи из шаблона доступны для fallback/`t`; псевдоязык «template» не показывается в переключателе языков).
- Обновить `i18n:extract` / связанные скрипты: целевые файлы для ключей игры и UI — `template.game.json` и `template.ui.json` (или согласованное расширение существующих скриптов), с сохранением поведения «не затирать уже введённые переводы».
- Упростить образцовый `game.ts`: убрать обязательные импорты шаблонных JSON, оставить только реальные языки автора.

## Capabilities

### New Capabilities

- `locale-templates`: Канонические `template.game.json` / `template.ui.json` под `src/locales`, импорт из ifKit, правила merge с авторскими локалями и исключение кода шаблона из списка выбора языка в UI.

### Modified Capabilities

- `i18n`: Требования к `getAvailableLanguages`, merge с внутренним шаблоном и к тому, что в настройках отображаются только «реальные» языки автора (плюс `sourceLanguage`), без фиктивного языка шаблона.
- `define-game-api`: Обновить норматив про импорт JSON в `game.ts` — шаблонные файлы подключает движок; автор передаёт только свои языки. Сценарии шаблонного репозитория (ifKit sample) отражают отсутствие импортов `template.*` в `game.ts`.
- `dev-tooling`: Скрипты извлечения SHALL обновлять `template.game.json` и `template.ui.json` (и не требовать от автора ручного добавления импорта ради попадания ключей в сборку).

## Impact

- `src/ifKit/` (engine / i18n init, возможно `engine.ts`), новые или изменённые JSON под `src/locales/`, `scripts/extract-*.mjs`, `package.json` при смене контракта скриптов, образцовый `src/game.ts`, документация в `README` / `docs` только если уже принято в проекте для i18n (в рамках change — по tasks).
