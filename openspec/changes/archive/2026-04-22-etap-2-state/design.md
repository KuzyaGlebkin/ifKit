## Context

Движок (`scenes.ts`) уже хранит ссылку на объект состояния в `currentState` и передаёт его в каждую сцену. `state.ts` и `snackbar.ts` — пустые заглушки. Нужно достроить три независимых слоя поверх существующей архитектуры, не меняя публичный API сцен.

## Goals / Non-Goals

**Goals:**
- Единый механизм клонирования (`jsonClone`) для истории и сохранений
- Локальные переменные сцены с автоматическим сбросом при `goto` и восстановлением при откате
- Snackbar: декларативная подписка через `watch` в `defineGame`, без прокси

**Non-Goals:**
- Реализация истории (undo/redo) — Этап 5
- Реализация слотов сохранений — Этап 5
- Реактивные вычисляемые значения (computed)
- Отслеживание вложенных объектов в `watch`

## Decisions

### 1. `jsonClone` — `JSON.parse(JSON.stringify(...))` везде

Альтернатива `structuredClone` поддерживает `Date`/`Map`/`Set` в памяти, но они не переживут `JSON.stringify` при сохранении в `localStorage`. Асимметрия между «работает в истории» и «ломается при сохранении» хуже, чем единое ограничение. Вывод: один механизм, одно правило.

Ограничение документируется: `GameState` должен быть JSON-совместимым деревом данных.

### 2. `sceneLocals` — пространство имён по `currentSceneKey`

Замыкания (`let x = 0` вне сцены) не сбрасываются при откате. WeakMap не подходит — ключ-строка. Выбран словарь `Record<sceneKey, Record<localKey, unknown>>` в `state.ts`. Доступ через `ctx.local(defaults)` — движок читает `currentSceneKey` из модульного замыкания `scenes.ts`.

Сброс: при `runGameLoop(newKey)`, если `newKey !== currentSceneKey`, локали старой сцены удаляются. При откате (Этап 5) локали восстанавливаются из снимка вместе с `state`.

### 3. `watch` — декларативный дифф без прокси

Прокси требует переоборачивания объекта при восстановлении из истории и усложняет отладку. Автодифф всех ключей не даёт контроля над форматом и показывает служебные изменения.

Решение: `watch?: WatchConfig<S>` в `GameConfig`. Движок снимает значения отслеживаемых ключей **до** колбэка, сравнивает `===` **после**, вызывает форматирующую функцию для изменившихся. Дифф только плоских примитивов — вложенные объекты в `watch` не поддерживаются в v1.

Дифф встраивается в `addAct` и `addGoto` через обёртку колбэка в `createContext`.

## Risks / Trade-offs

- [JSON-ограничение молчит] Если автор положит `Date` в состояние, она превратится в строку при первом клонировании — без исключения. → Митигация: JSDoc-комментарий на `GameConfig.state` + раздел в документации.
- [Плоский дифф в watch] Автор не может отслеживать `state.inventory.length` напрямую — только `state.inventoryCount`. → Принято для v1, можно расширить позже.
- [sceneLocals и Этап 5] Снимок для истории должен включать `sceneLocals[currentSceneKey]`. Интерфейс `getSceneLocalSnapshot` / `restoreSceneLocal` нужно заложить сейчас. → Экспортируем их из `state.ts` с пометкой `@internal`.
