## Why

Сейчас `settings.language` и `t` влияют на текст в сценах, но подписи движка (модалка настроек, тулбар, модалка сохранений и т.д.) остаются на фиксированном языке, а `document.documentElement` не отражает активную локаль. Вспомогательные технологии полагаются на `lang` у `<html>`, поэтому при смене языка игры объявляемая локаль страницы должна совпадать с фактическим языком UI.

## What Changes

- Ввести **словарь системного (движкового) UI-текста** в составе объекта локализации: для каждого кода языка — отдельный набор строк по стабильным ключам (не смешивать с произвольными ключами `t` для сцен), чтобы движок мог переводить интерфейс согласованно с игрой.
- **Резолвить** активный BCP-47 (или primary language subtag) так же, как для `t` (включая режим «Авто»), и:
  - обновлять **`<html lang="…">`** при старте и при любом изменении настройки языка;
  - перерисовывать/обновлять строки **всего** движкового UI, завязанного на настройки (как минимум модалка настроек, при необходимости — тулбар, модалка сохранений, snackbar, aria-label там, где они задаются движком).
- **Начальный** `lang` в разметке (шаблон `index.html`) согласовать с поведением движка: либо нейтральное значение до первого apply, либо документировать, что движок перезаписывает при инициализации.

**BREAKING**: Авторы с `locales` должны предоставлять в конфиге переводы системных строк для каждого поддерживаемого языка (или движок определяет встроенные fallback-наборы — см. design). Уточнение по обратной совместимости — в `design.md`.

### Modified Capabilities

- `i18n`: хранение и выдача **системного** словаря; связь `setLanguage` / resolved lang с обновлением `document.documentElement.lang`; единая модель «активного языка» для сцен и UI.
- `define-game-api`: расширение контракта `locales` (или соседнее поле), включающее **ui** / **system** подсловарь по языкам.
- `user-settings`: применение настроек к DOM включает **синхронизацию `lang` на `<html>`** с разрешённым языком (вызов при `applySettings` / при смене языка — деталь в design).
- `settings-modal` (и при необходимости смежные: `saves-modal`, `ui-buttons`, `ui-zones`): **все** пользовательские строки движка в настраиваемых зонах берутся из системного словаря и обновляются при смене языка, а не из захардкоженного одного языка.

## Impact

- `src/ifKit/i18n.ts` — API для системных строк; обновление `lang`.
- `src/ifKit/engine.ts` — инициализация/слияние ui-словаря с `locales`.
- `src/ifKit/settings-modal.ts`, модалки/тулбар — убрать жёстко заданные строки, подписаться на смену языка.
- `src/index.html` — начальный `lang` (и при необходимости `title` — вне scope, если не трогаем).
- Типы: `Locales` или `GameConfig` в публичном API.
- OpenSpec-спеки: дельты перечисленных capability.
