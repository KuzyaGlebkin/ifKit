## ADDED Requirements

### Requirement: Системные подписи модалки настроек

Все **видимые** пользователю строки, задаваемые **движком** в модалке настроек (заголовок, подписи секций «Тема», «Шрифт», «Акцент», «Музыка», «Звуки», «Новый текст» / сброс прочтения, кнопки «Сбросить настройки», «Экспорт», «Импорт», подписи пресетов темы/акцента, подпись «Авто» в языковой группе), SHALL получаться **через** системный слой локализации (например `u(ключ)`), **а не** жёстко кодируемым русским/английским текстом в исходниках. Исключение: **коды** языков в кнопках `ru`/`en`/… остаются **как есть** (не переводы).

#### Scenario: Смена языка обновляет подписи настроек

- **WHEN** у игры есть `locales` с переводом системных ключей для двух языков и игрок переключил язык
- **THEN** заголовок и подписи модалки настроек (после обновления UI) **соответствуют** выбранной локали, а не остаются на прежнем единственном языке

#### Scenario: Секция «Язык» с локализованной подписью

- **WHEN** отображается секция выбора языка (2+ языка) и `u` для ключа подписи «Язык» (или аналог) возвращает локализованный текст
- **THEN** `aria-labelledby` / видимая подпись **используют** этот перевод, согласованный с `document.documentElement.lang`

### Requirement: Перерисовка модалки при смене resolved языка

При изменении resolved языка (в т.ч. кнопки языка, «Сбросить», импорт настроек) движок SHALL **обновлять** отображаемый текст **системных** подписей модалки, если она **создана** в DOM, без обязательного закрытия модалки (реализация: полная пересборка HTML модалки или точечное обновление — на усмотрение, поведение нормативно).

#### Scenario: Язык сменили при открытой модалке

- **WHEN** модалка настроек **открыта** и игрок нажал другой язык
- **THEN** **видимые** движковые подписи (хотя бы заголовок и кнопки, не на código языка) **обновляются** до новой локали
