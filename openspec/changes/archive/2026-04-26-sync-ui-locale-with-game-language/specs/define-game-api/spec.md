## ADDED Requirements

### Requirement: Локали с вложенными game и ui

`defineGame()` SHALL принимать `locales` в форме, позволяющей для **каждого** кода языка задать **как** словарь игровых строк для `t` (прежняя семантика), **так** и словарь **системных** строк (ключи UI). Допустимая форма: для каждого кода — объект с полями `game` и `ui` (типы уточняются в типах публичного API). Движок SHALL **нормализовать** устаревший формат «плоский `Record` на язык» в «только `game`» и подмешивать `ui` из встроенных умолчаний, как определено в design.

#### Scenario: Плоский словарь остаётся валидным

- **WHEN** автор оставил прежний формат `locales: { en: { "Привет": "Hello" } }` без вложенности `game`/`ui`
- **THEN** `t` для игровых строк **ведёт себя** как раньше, а `u` **использует** встроенные системные строки для `en` (и других задействованных языков) где автор не дал `ui`

#### Scenario: Расширенный формат с ui

- **WHEN** автор передал `locales: { en: { game: { ... }, ui: { "settings.title": "Settings" } } }`
- **THEN** `u('settings.title')` при активном `'en'` возвращает `"Settings"` (при согласовании реального ключа в реализации)

### Requirement: fullLocales и system ui для sourceLanguage

Если задано `sourceLanguage` и `fullLocales` строятся с пустым слоем для кода оригинала, движок SHALL обеспечивать, чтобы **системные** строки (`u`) **для** этого кода **не** исчезали: пустой авторский `ui` **должен** сливаться с встроенным словарём, аналогично остальным языкам.

#### Scenario: sourceLanguage и пустой game-слой

- **WHEN** `sourceLanguage: 'ru'` и для `'ru'` нет игровых переводов, но `u` вызывается
- **THEN** движок **не** оставляет подписи модалки пустыми из‑за «пустой» игровой локали
