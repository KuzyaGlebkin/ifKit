## Context

Панель `#controls` в `src/index.html` — последовательность: логотип, четыре `<button>` с фиксированными `id`. В `style.css` для `#controls` заданы `display: flex`, `flex-wrap: wrap`, без разделения на «левый бренд / правая группа действий». Скринридеры объявляют кнопки по одной, без групповой семантики.

## Goals / Non-Goals

**Goals:**

- Визуально: логотип `.ifk-brand-logo` слева на полосе; блок из четырёх кнопок выровнен по правому краю навигации при достаточной ширине.
- Разметка: четыре кнопки обёрнуты в нативный `<ul>` с четырьмя `<li>`, без маркеров, чтобы AT могли использовать семантику списка.
- Сохранить `id` кнопок, порядок фокуса Tab, медиазапросы для подписей и скрытия логотипа на очень узкой ширине.

**Non-Goals:**

- Смена порядка кнопок, новые кнопки, изменение иконок Lucide или логики `refreshToolbarI18n`.
- Локализация `aria-label` на `<ul>` (при необходимости — отдельная задача вместе с i18n shell).

## Decisions

1. **Обёртка `<ul class="…">` вокруг кнопок** — нативный список даёт предсказуемую роль для AT; `list-style: none` и нулевые отступы в CSS убирают маркеры. Альтернатива `role="toolbar"` отклонена: пользователь явно запросил «списком».
2. **Выравнивание вправо** — на `#controls` оставить `display: flex; align-items: center; flex-wrap: wrap`; на обёртке списка задать `margin-inline-start: auto` и `display: flex` с `gap`, повторяя текущий визуальный зазор между кнопками. При переносе на вторую строку (узкий viewport) блок со списком может занять строку целиком — приемлемый компромисс; при необходимости уточнить в follow-up.
3. **Класс обёртки** — например `.ifk-controls-actions`, чтобы не завязаться на сырой тег в селекторах движка.

## Risks / Trade-offs

- **[Risk] Старые селекторы вида `#controls button`** — продолжают матчить кнопки внутри `li`; риск минимален.
- **[Risk] Перенос строки** — при `flex-wrap` логотип и список могут оказаться на разных строках; митигация: `align-items: center` и осознанный `gap`; на очень узком экране поведение как сейчас (частичный перенос).
- **[Trade-off] Не все скринридеры объявляют «список из N»** для горизонтальных списков — семантика всё равно полезна для структуры дерева доступности.

## Migration Plan

Изменения только в шелле и стилях; откат — revert коммита. После merge — `openspec archive` по процессу проекта.
