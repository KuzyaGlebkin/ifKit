## ADDED Requirements

### Requirement: Семантический список глобальных кнопок `#controls`

Кнопки `#btn-undo`, `#btn-redo`, `#btn-saves`, `#btn-settings` SHALL быть сгруппированы в один нативный список: контейнер `<ul>` с четырьмя дочерними `<li>`, каждый `<li>` SHALL содержать ровно одну из указанных кнопок с неизменёнными `id`. Визуально список SHALL NOT показывать маркеры (`list-style: none` и согласованные отступы). Порядок элементов списка SHALL совпадать с порядком Tab по кнопкам (undo → redo → saves → settings). Контейнер `<ul>` / `<li>` SHALL NOT быть ловушкой фокуса: только кнопки остаются в последовательной навигации Tab.

#### Scenario: Разметка списка

- **WHEN** браузер загружает `src/index.html`
- **THEN** внутри `<nav id="controls">` существует один `<ul>`, содержащий четыре `<li>`, и в каждом `<li>` ровно одна кнопка с `id` из набора `btn-undo`, `btn-redo`, `btn-saves`, `btn-settings`

#### Scenario: Tab только по кнопкам

- **WHEN** пользователь последовательно нажимает Tab, перемещаясь по глобальной панели после логотипа
- **THEN** фокус переходит между четырьмя кнопками в порядке разметки без остановки на `<ul>` или `<li>` как на отдельных tab-stop

#### Scenario: Согласованность с узким экраном

- **WHEN** ширина viewport не более 600px (режим только иконок на кнопках)
- **THEN** требования «Узкий экран — только иконки на `#controls`» и «Скринридер на узком экране» из базовой спеки `ui-buttons` по-прежнему выполняются для кнопок внутри списка
