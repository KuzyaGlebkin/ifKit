## Why

Панель `#controls` сейчас — один flex-ряд с переносом: логотип и кнопки визуально «текут» вместе, без явного разделения «бренд слева — действия справа». Для скринридеров кнопки звучат как отдельные элементы подряд без группировки. Нужна предсказуемая шапка: логотип слева, блок команд прижат к правому краю, с семантикой списка для вспомогательных технологий.

## What Changes

- Разметка `#controls`: `.ifk-brand-logo` остаётся первым блоком слева; кнопки undo/redo/saves/settings группируются в семантический список (`<ul>` / `<li>`), визуально без маркеров.
- CSS: раскладка на всю ширину полосы — группа кнопок выравнивается по правому краю (`margin-inline-start: auto` или эквивалент), с сохранением порядка кнопок и существующих медиаправил (узкий экран, скрытие подписей).
- Сохранить текущие `id` кнопок и поведение фокуса/модалок; не ломать требования про возврат фокуса на `#btn-saves` / `#btn-settings`.

### Modified Capabilities

- `app-branding`: уточнить раскладку панели — логотип слева, группа кнопок справа на полосе навигации; сохранить акцентный цвет и правила фокуса для логотипа.
- `ui-buttons`: добавить требование семантического списка для четырёх кнопок `#controls` без изменения порядка Tab и без визуальных маркеров списка.
- `project-scaffold`: уточнить сценарий скелета `index.html` — допустима обёртка кнопок в `<ul>/<li>` при сохранении тех же `id`.
