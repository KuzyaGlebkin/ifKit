## Context

Слоты сохраняют `label` как обрезок `textContent` `#scene-content`. Кнопки act и goto рендерятся в `#scene-acts` и `#scene-gotos` (`renderer.ts`). Карточка слота в `saves-modal.ts` собирает `.ifk-slot-info` и горизонтальный `.ifk-slot-actions`. Старые записи в `localStorage` не содержат новых полей.

## Goals / Non-Goals

**Goals:**

- При **ручном** сохранении в именной слот записывать текст первой подписи первого act и первого goto из текущего DOM.
- Показывать эти строки в `.ifk-slot-info` для занятых слотов, не ломая старые сохранения.
- Кнопки «Загрузить», «Сохранить», «Удалить» в обычном режиме — вертикальный столбец с полной шириной в колонке действий (в пределах карточки).

**Non-Goals:**

- Локализация отдельных подписей «Act:» / «Goto:» в UI (если понадобится, отдельная задача и ключи i18n).
- Изменение порядка фокуса Tab или логики подтверждений, кроме визуальной раскладки.

## Decisions

1. **Имена полей в `SaveSlot`:** `actPreview` и `gotoPreview`, тип `string`. Пустая строка, если контейнер пуст или элемента нет. При чтении старых JSON отсутствие ключей трактуется как `''`.

2. **Извлечение текста:** общая функция: первый дочерний элемент-кандидат в `#scene-acts` / `#scene-gotos` (кнопка `.btn-act` / ссылка `.btn-goto` как в разметке движка), `textContent`, `trim()`, обрезка до **80** символов по аналогии с `label`. Если селекторы изменятся, привязка к существующим классам из `renderer.ts` предпочтительнее полного обхода всех узлов.

3. **Автослот:** `autoSave` вызывается до `runGameLoop` при `goto`, поэтому DOM ещё показывает **предыдущую** сцену — не извлекать act/goto в этом пути без перестановки вызова. В объекте автослота поля `actPreview` и `gotoPreview` SHALL быть пустыми строками; карточка автослота по-прежнему без превью сцены/act/goto. **Именной** слот: заполнять оба поля в `saveToSlot` из актуального DOM. Превью act/goto в `.ifk-slot-info` только у занятых **именных** слотов (где показывается `label`).

4. **Вертикальные кнопки:** `.ifk-slot-actions { flex-direction: column; align-items: stretch; }` для основного режима; кнопки `width: 100%` или `align-self: stretch`. Режим подтверждения: текст и «Да»/«Нет» тоже колонкой или блоком, без горизонтальной «лесенки», сохраняя читаемость.

5. **Доступность:** при добавлении строк в `ifk-slot-info` убедиться, что `aria-labelledby` по-прежнему озвучивает номер и пользовательскую подпись слота; дополнительные строки станут частью текстового дерева группы (дополнительные `span` без отдельных id, если не требуется).

## Risks / Trade-offs

- **[Risk]** Длинные подписи act/goto переполняют карточку → **Mitigation:** ellipsis и `min-width: 0` на колонке info (уже есть на `.ifk-slot-info`).
- **[Risk]** В момент сохранения DOM пуст (нет act/goto) → **Mitigation:** пустые строки, UI не показывает лишних пустых блоков или показывает компактно.
- **[Trade-off]** Превью фиксируются на момент сохранения и не отражают изменения текста в игре до пересохранения — ожидаемо.

## Migration Plan

- Код чтения слотов: `actPreview ?? ''`, `gotoPreview ?? ''` при десериализации.
- Откат: вернуть CSS и разметку; старые поля в JSON безвредны.

## Open Questions

- Нужны ли префиксы в UI («Act: …») — по умолчанию нет, только текст; при тестировании при необходимости добавить `UI.*` ключи.
