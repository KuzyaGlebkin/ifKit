## Why

Карточка слота в модалке сохранений даёт мало контекста: видны только номер, время и отрывок текста сцены. Игроку сложнее отличить слоты, где важны доступные действия (act) и переходы (goto). Кнопки «Загрузить», «Сохранить», «Удалить» сейчас в один ряд; вертикальная укладка улучшает читаемость и согласуется с узкими карточками.

## What Changes

- Расширить метаданные слота сохранения: при сохранении фиксировать **первую подпись первой кнопки act** (`#scene-acts`) и **первую подпись первого goto** (`#scene-gotos`) так же, как уже фиксируется отрывок текста сцены (`label`).
- В блоке `.ifk-slot-info` для занятых слотов отображать эти две строки (с обрезкой/ellipsis при необходимости), с корректным поведением для старых сохранений без новых полей.
- Разметка/стили: контейнер `.ifk-slot-actions` для режима обычных кнопок SHALL располагать «Загрузить», «Сохранить», «Удалить» **вертикально** (колонка); подтверждения «Да»/«Нет» остаются согласованными с новым направлением.
- Автослот: в данных хранить пустые `actPreview`/`gotoPreview` (вызов `autoSave` до рендера новой сцены не позволяет честно снять подписи); UI автослота без этих строк.

## Capabilities

### New Capabilities

- _(нет)_

### Modified Capabilities

- `save-slots`: тип `SaveSlot` и правила записи при сохранении/автосохранении — добавить поля превью act/goto и их заполнение из DOM.
- `saves-modal`: отображение `.ifk-slot-info`; раскладка `.ifk-slot-actions` (вертикальный стек кнопок); при необходимости сценарии доступности для новых строк.

## Impact

- `src/ifKit/saves.ts` — тип `SaveSlot`, `saveToSlot`, `autoSave` (явные пустые `actPreview`/`gotoPreview` для auto).
- `src/ifKit/saves-modal.ts` — разметка карточки слота, возможно `aria-labelledby` при добавлении строк.
- `src/style.css` — flex-direction / ширина кнопок в `.ifk-slot-actions`, стили строк превью act/goto.
- Обратная совместимость: десериализация старых JSON из `localStorage` без новых ключей.
