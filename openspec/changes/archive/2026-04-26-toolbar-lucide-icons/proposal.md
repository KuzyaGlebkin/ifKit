## Why

Кнопки верхней панели и действий в модалке сохранений полагаются только на текст; на узкой ширине экрана подписи занимают много места и хуже считываются как быстрые действия. В проекте уже подключена библиотека **lucide** — имеет смысл дать кнопкам узнаваемые пиктограммы и на узких viewport показывать только иконки, сохраняя доступное имя действия для вспомогательных технологий.

## What Changes

- Добавить иконки **Lucide** к кнопкам `#controls`: «Назад» → `Undo`, «Вперёд» → `Redo`, «Сохранения» → `Save`, «Настройки» → `Settings`.
- В модалке сохранений: «Загрузить» → `ArrowUpFromLine`, «Сохранить» → `ArrowDownToLine`, «Удалить» → `Trash` (и согласованное оформление в режимах подтверждения — по дизайну).
- На **узких экранах** (breakpoint по дизайну): видимый текст на этих кнопках скрывается, остаются иконки; для скринридеров и подсказок SHALL сохраняться понятные подписи (`aria-label` / `title` или эквивалент).
- Стили: иконки визуально согласованы с текущей типографикой и отступами панели/модалки.

## Capabilities

### New Capabilities

(none)

### Modified Capabilities

- `ui-buttons`: требования к разметке и отображению кнопок `#controls` с иконками Lucide и адаптивным режимом «только иконка» на узкой ширине; доступность подписи при скрытом тексте.
- `saves-modal`: те же принципы для кнопок слота «Загрузить» / «Сохранить» / «Удалить» (и при необходимости кнопок подтверждения «Да»/«Нет», если затрагиваются макетом).

## Impact

- `package.json` / зависимости: **без новых** — `lucide` уже в проекте.
- `src/index.html` — разметка кнопок `#controls` (обёртки для иконки + текст).
- `src/style.css` — медиазапросы и классы для icon-only на узкой ширине.
- `src/ifKit/scenes.ts` или общий хелпер — при необходимости инициализация/обновление иконок для undo/redo (если не статичный SVG).
- `src/ifKit/saves.ts` (или модуль модалки сохранений) — разметка кнопок слотов с иконками.
- Импорт иконок из `lucide` (tree-shaking дружелюбные именованные импорты).
