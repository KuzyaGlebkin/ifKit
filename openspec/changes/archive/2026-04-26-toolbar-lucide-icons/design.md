## Context

В `src/index.html` кнопки `#controls` — только текст («Назад», «Вперёд», …). Модалка сохранений собирается в коде (`saves.ts` / связанный модуль) с текстовыми кнопками «Загрузить», «Сохранить», «Удалить». Зависимость **`lucide`** уже в `package.json`; в исходниках импортов пока нет. Пакет экспортирует **IconNode** и **`createElement(iconNode, attrs)`** для получения `SVGElement` — удобно вставлять иконку первым ребёнком кнопки без отдельного рантайма `createIcons` по всему документу.

## Goals / Non-Goals

**Goals:**

- У каждой указанной кнопки — иконка Lucide с фиксированным соответствием: Назад `Undo`, Вперёд `Redo`, Сохранения `Save`, Настройки `Settings`; в слотах — `ArrowUpFromLine` (Загрузить), `ArrowDownToLine` (Сохранить), `Trash` (Удалить).
- На широком viewport: иконка + текущий видимый текст (как сейчас по смыслу).
- На узком viewport: **видимый** текст скрыт, остаётся иконка; имя действия для AT — через `aria-label` на `<button>` (и при необходимости `title` для указательных устройств), текст в разметке MAY оставаться для совместимости, но скрыт визуально (`sr-only` / эквивалент), либо дублируется только в `aria-label` — выбрать один согласованный подход при реализации (предпочтительно: видимая подпись в DOM там, где она не скрыта, и полный `aria-label` в режиме icon-only, чтобы не терять локализацию).
- Размер иконки и отступы согласовать с `#controls` и карточками слотов (единый класс вроде `ifk-icon-btn` / `ifk-lucide-icon`).

**Non-Goals:**

- Смена поведения undo/redo/saves/settings и сценариев модалок (кроме разметки/стилей/подписей).
- Иконки для `act` / `goto` или для кнопок «Да»/«Нет» в подтверждениях — только если понадобится для выравнивания сетки; по умолчанию без новых пиктограмм.
- Замена `lucide` на другой пакет.

## Decisions

1. **API Lucide:** именованные импорты узлов иконок + `createElement` из `lucide` (как в официальной документации пакета `lucide`). Альтернатива `createIcons` по `data-lucide` — меньше контроля над порядком детей в динамически пересобираемой модалке слотов.

2. **Breakpoint для icon-only:** **`max-width: 600px`** для панели `#controls` и кнопок действий в модалке сохранений (отдельно от порога **480px** для полноэкранной модалки и колонок настроек).

3. **Разметка кнопки:** обёртка для иконки с `aria-hidden="true"`, отдельный span для подписи с классом, который скрывается медиазапросом. Иконку вставлять/обновлять в `DOMContentLoaded` / при создании модалки один раз на кнопку (не пересоздавать при каждом ререндере слота без нужды).

4. **Локализация:** подписи для `aria-label` в режиме icon-only SHALL совпадать по смыслу с текущими строками UI (русские строки или ключи i18n, если движок уже тянет строки из одного места — использовать тот же источник).

## Risks / Trade-offs

- **[Risk] Дублирование подписи** (экранный текст + `aria-label`) при широком экране → **Mitigation:** на широком экране не задавать избыточный `aria-label`, если видимый текст остаётся содержимым кнопки; в режиме icon-only задать `aria-label` явно.
- **[Risk] Перерисовка модалки сохранений** сбрасывает DOM с иконками → **Mitigation:** централизовать создание кнопки слота в одной функции, которая каждый раз добавляет иконку + текст.

## Migration Plan

Не требуется: изменения только в фронтенд-разметке/стилях/инициализации; откат — revert коммита.
