## ADDED Requirements

### Requirement: Тип Scenes обеспечивает типизацию функций-сцен
Движок SHALL экспортировать тип `Scenes<S>`, представляющий объект, где каждое значение — функция вида `(state: S) => void`.

#### Scenario: Тип принимает корректный объект сцен
- **WHEN** автор объявляет `const scenes: Scenes<GameState> = { start: (state) => { P('Старт') } }`
- **THEN** TypeScript не выдаёт ошибок типизации

#### Scenario: Тип отклоняет функцию с неверной сигнатурой
- **WHEN** автор объявляет сцену как `start: () => 'строка'`
- **THEN** TypeScript выдаёт ошибку о несоответствии типа

### Requirement: goto выполняет переход к сцене по ключу
Движок SHALL предоставлять функцию `goto(key)`, которая запускает полный игровой цикл для указанной сцены: очищает буфер, вызывает функцию сцены, сбрасывает буфер в DOM.

#### Scenario: Переход к существующей сцене
- **WHEN** автор вызывает `goto('clearing')` и такая сцена зарегистрирована
- **THEN** функция `clearing` вызывается, и её вывод появляется в DOM

#### Scenario: Переход к несуществующей сцене вызывает ошибку
- **WHEN** во время выполнения вызывается `goto('nonexistent')` с ключом, которого нет в объекте сцен
- **THEN** движок выбрасывает ошибку с информативным сообщением (имя отсутствующей сцены)

### Requirement: Ключи сцен проверяются статически на этапе компиляции
Параметр `key` функции `goto` SHALL иметь тип `keyof typeof scenes`, так что передача несуществующего ключа приводит к ошибке TypeScript.

#### Scenario: Компиляция с валидным ключом проходит без ошибок
- **WHEN** автор пишет `goto('start')` и ключ `start` существует в `scenes`
- **THEN** TypeScript компилирует файл без ошибок

#### Scenario: Компиляция с невалидным ключом завершается ошибкой
- **WHEN** автор пишет `goto('nonexistant')` и такого ключа нет в `scenes`
- **THEN** TypeScript выдаёт ошибку: аргумент типа `'nonexistant'` не присваивается параметру

### Requirement: Игровой цикл состоит из трёх детерминированных шагов
При каждом переходе к сцене движок SHALL выполнять строго по порядку: (1) очистить буфер, (2) вызвать функцию сцены, (3) сбросить буфер в DOM.

#### Scenario: Контент предыдущей сцены не просачивается
- **WHEN** сцена A рендерит `P('Сцена A')`, затем вызывается `goto('b')`, где сцена B рендерит `P('Сцена B')`
- **THEN** DOM содержит только `<p>Сцена B</p>`, без `<p>Сцена A</p>`

#### Scenario: Исключение в функции сцены не оставляет частичный контент в DOM
- **WHEN** функция сцены выбрасывает ошибку после записи в буфер
- **THEN** DOM не обновляется (буфер не сбрасывается в случае ошибки)
