## Why

Этап 0 создал скаффолдинг проекта, но движок пуст: нет буфера рендеринга, нет теговых функций, нет механизма переходов между сценами. Без этого ядра `defineGame()` нельзя реализовать — она не знает, как рендерить контент и переключать сцены.

## What Changes

- Появляется буферная система рендеринга: строковый буфер, в который теговые функции пишут HTML, и механизм его сброса в DOM
- Реализуются теговые функции двух видов: императивные (`P`, `H1`, `Div` и др.) — пишут в буфер; функциональные (`p`, `h1`, `div` и др.) — возвращают HTML-строку
- Вводится тип `Scenes<S>` и функция `registerScenes()`, обеспечивающая type-safe ключи через `keyof typeof scenes`
- Реализуется `goto(key)` — переход к сцене с проверкой ключа на этапе компиляции
- Появляется базовый игровой цикл: `clearBuffer → runScene → flushToDOM`
- `defineGame()` связывается с новым движком: регистрирует сцены, запускает игровой цикл, вызывает `static` при каждом переходе

## Capabilities

### New Capabilities

- `render-buffer`: Буфер рендеринга — хранение накопленного HTML и его применение к DOM
- `tag-functions`: Теговые функции — императивные (`P`, `H1`, `Img` и др.) и функциональные (`p`, `h1`, `img` и др.)
- `scene-engine`: Движок сцен — тип `Scenes`, регистрация сцен, функция `goto`, игровой цикл

### Modified Capabilities

- `define-game-api`: `defineGame()` теперь реально запускает движок: передаёт `scenes` в `registerScenes()`, вызывает `static` через буфер, инициирует игровой цикл

## Impact

- Новые файлы: `src/ifKit/renderer.ts`, `src/ifKit/scenes.ts`, `src/ifKit/transitions.ts`
- Обновляется `src/ifKit/index.ts`: экспортирует теговые функции и `goto` для использования в `game.ts` и `scenes.ts`
- Обновляется `src/ifKit/index.ts`: реализует `defineGame()` поверх нового движка
- Нет новых внешних зависимостей — только нативные DOM API и TypeScript
