## Context

Контейнер snackbar создаётся в `initSnackbar` (`src/ifKit/snackbar.ts`), элементы сообщений — дочерние `div.snackbar-item` в `#snackbar-container`. Сейчас у контейнера нет роли и live region; скринридеры не объявляют появляющийся текст.

## Goals / Non-Goals

**Goals:**

- Один предсказуемый статусный регион для всех уведомлений из очереди snackbar.
- Объявления в режиме `polite`, без прерывания текущей речи AT.

**Non-Goals:**

- Смена визуальной стилизации, длительности показа или логики `watch`.
- `aria-live="assertive"` (слишком агрессивно для игровых подсказок).
- Отдельный live region на каждую плашку (усложняет дерево и не требуется для текущей модели «стек в одном контейнере»).

## Decisions

1. **Где вешать ARIA** — на `#snackbar-container` сразу после `createElement`, в `initSnackbar` (и при повторной инициализации контейнер уже есть — атрибуты уже на месте). Не дублировать `role="status"` на каждом `.snackbar-item`, чтобы не плодить регионы и не спорить с одним родителем-статусом.

2. **`role="status"` + `aria-live="polite"`** — соответствует паттерну «тост / статус» в [APG](https://www.w3.org/WAI/ARIA/apg/patterns/alert/): статус менее навязчив, чем `alert`. Явно задать оба: `role="status"` подразумевает live по умолчанию, но явный `aria-live="polite"` фиксирует намерение для ревью и для движков, где эвристики различаются.

3. **`aria-atomic`** — не выставлять; по умолчанию `false`, что удобнее при поочерёдном добавлении нескольких `.snackbar-item` за один кадр (AT чаще фокусируются на новом фрагменте, а не перечитывают весь контейнер).

## Risks / Trade-offs

- **[Различие AT при нескольких быстрых вставках]** → Приёмлемо для v1; при жалобах можно рассмотреть одну замену `textContent` на контейнере или очередь с задержкой объявления.
- **[Пустой контейнер в дереве доступности]** → Контейнер остаётся пустым между тостами; это нормально для `role="status"` без лишнего `aria-label`.

## Migration Plan

Не требуется: изменение только в клиентской разметке, обратная совместимость с сохранениями и API игры не затрагивается.
