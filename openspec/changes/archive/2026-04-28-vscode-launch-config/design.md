## Context

Проект — Vite-приложение с корнем `src/` (`vite.config.ts`), npm-скрипты `dev`, `tauri:dev`, `build` уже определены в `package.json`. Редактор VS Code / Cursor использует те же файлы `.vscode/*`, что и VS Code.

## Goals / Non-Goals

**Goals:**

- Дать автору один клик для запуска веб-сборки в браузере с рабочим dev-сервером и точкой подключения отладчика.
- По возможности покрыть сценарий нативной оболочки через `tauri:dev` отдельной конфигурацией (без обязательного compound-сценария, если он усложняет поддержку).

**Non-Goals:**

- Настройка отладки Rust-сайда Tauri и `lldb`/`cppvsdbg`.
- Замена документации по установке Node, Rust или расширений VS Code.
- Обязательная поддержка других IDE (IntelliJ, WebStorm) в рамках этого change.

## Decisions

1. **Файлы `.vscode/launch.json` и `.vscode/tasks.json`**  
   Разделение: `tasks.json` поднимает долгоживущие процессы (`npm run dev`, при необходимости `tauri:dev`) с `isBackground: true` и `problemMatcher`, чтобы VS Code дождался готовности сервера перед открытием браузера.  
   *Альтернатива:* только `launch.json` без задач — отклонена: без фоновой задачи отладчик не синхронизируется со стартом Vite.

2. **URL для веб-режима**  
   Использовать `http://localhost:5173/` (порт по умолчанию Vite; в конфиге порт явно не переопределён). Если в будущем порт изменится, обновляют и `launch.json`, и спецификацию.  
   *Альтернатива:* переменная `${env:VITE_PORT}` — отклонена как избыточная для текущего репозитория.

3. **Тип отладчика для браузера**  
   Использовать встроенный JavaScript Debugger (конфигурации вида `pwa-chrome` / `chrome` в зависимости от версии VS Code), `request: launch`, `url` на dev-сервер.  
   *Альтернатива:* только «attach» к уже открытому Chrome — хуже для «одного клика».

4. **Tauri**  
   Отдельная конфигурация запуска, вызывающая задачу `npm run tauri:dev` (фоновый процесс с подходящим `problemMatcher` по логу готовности Vite/Tauri). Без отладки Rust.  
   *Альтернатива:* не добавлять Tauri в launch — отклонена: скрипт уже в проекте, авторы шаблона ожидают симметрии с `dev`.

## Risks / Trade-offs

- **[Risk]** Хрупкий `problemMatcher` при смене формата лога Vite → ломается ожидание готовности задачи.  
  **Mitigation:** использовать распространённый шаблон для Vite 5/6 и при обновлении CLI проверять вывод `npm run dev`.

- **[Risk]** Порт 5173 занят другим процессом.  
  **Mitigation:** стандартное поведение Vite; в спеке зафиксировать согласование URL с дефолтом.

- **[Trade-off]** Отладка только front-end; нативные краши Tauri без JS-стека остаются вне сценария.

## Migration Plan

Не применимо: добавляются только файлы редактора и дельта спеки. Откат — удаление `.vscode/launch.json` и `tasks.md` и откат дельты `dev-tooling`.
