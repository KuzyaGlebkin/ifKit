## ADDED Requirements

### Requirement: Конфигурации запуска VS Code для разработки

Репозиторий SHALL содержать в каталоге `.vscode/` файлы `launch.json` и `tasks.json`, согласованные с npm-скриптами проекта.

`tasks.json` SHALL определять фоновые задачи для:

- запуска веб dev-сервера (`npm run dev`);
- при наличии скрипта в `package.json` — запуска десктопной разработки Tauri (`npm run tauri:dev`).

`launch.json` SHALL содержать как минимум одну конфигурацию, которая:

- запускает или присоединяет отладчик к браузеру (встроенный JS Debugger для Chrome/Chromium);
- открывает приложение по URL dev-сервера Vite по умолчанию (`http://localhost:5173/`, если в `vite.config` не задан другой порт — в этом случае URL SHALL соответствовать фактическому порту в конфигурации);
- использует предстартовую задачу из `tasks.json`, чтобы dev-сервер был поднят до открытия браузера.

`launch.json` SHOULD содержать отдельную конфигурацию для сценария `tauri:dev`, если в проекте определён скрипт `tauri:dev`, с привязкой к соответствующей фоновой задаче.

Конфигурации SHALL быть валидным JSON и не ссылаться на несуществующие имена задач.

#### Scenario: Веб-разработка из редактора

- **WHEN** разработчик открывает проект в VS Code или Cursor и выбирает конфигурацию запуска для веб dev-сервера
- **THEN** редактор поднимает `npm run dev`, дожидается готовности сервера и открывает отладку в браузере по URL dev-приложения

#### Scenario: Согласованность с npm-скриптами

- **WHEN** разработчик сравнивает задачи в `tasks.json` с `package.json`
- **THEN** используемые `script`-имена совпадают с существующими ключами (`dev`, и при наличии конфигурации Tauri — `tauri:dev`)

#### Scenario: Наличие обоих файлов

- **WHEN** разработчик просматривает каталог `.vscode/`
- **THEN** присутствуют оба файла `launch.json` и `tasks.json`
