## Why

Разработчикам удобнее запускать и отлаживать проект из VS Code / Cursor одной командой из панели Run and Debug, без ручного ввода `npm run dev` и открытия URL. Сейчас в репозитории нет готовых конфигураций запуска для редактора.

## What Changes

- Добавить каталог `.vscode/` с `launch.json` (и при необходимости `tasks.json`) для согласованного запуска с существующими npm-скриптами (`dev`, опционально `tauri:dev`).
- Зафиксировать требование в спецификации `dev-tooling`, чтобы шаблон проекта оставался воспроизводимым для авторов.

## Capabilities

### New Capabilities

- (нет — поведение продукта для игрока не меняется)

### Modified Capabilities

- `dev-tooling`: добавить требование о наличии конфигураций запуска VS Code для веб-разработки (и при согласовании с дизайном — для Tauri), с привязкой к `npm run dev` / `tauri:dev`.

## Impact

- Новые файлы только в `.vscode/` (и дельта `openspec/specs/dev-tooling` через change).
- Код приложения, зависимости и CI не затрагиваются.
