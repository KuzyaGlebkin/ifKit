## Context

В проекте ifKit управление сценой и модалками реализовано на чистом DOM (`src/index.html`, `src/ifKit/scenes.ts`, `src/ifKit/settings-modal.ts`, `src/ifKit/saves-modal.ts`). Кнопки `#btn-undo` / `#btn-redo` сейчас переводятся в `disabled`, из‑за чего браузер исключает их из цикла Tab. Модалки следуют общему паттерну backdrop + панель, но без удержания фокуса, что позволяет открыть вторую модалку с клавиатуры. Секции «Музыка»/«Звуки» используют `<label for="…">`, секция языка — группа кнопок без такого же явного контекста. После `runGameLoop` фокус не управляется и может оставаться на элементе предыдущей сцены.

## Goals / Non-Goals

**Goals:**

- Предсказуемая клавиатурная и скринридерная навигация: история в Tab, модалки не допускают «двойного» открытия с Tab, смена сцены ведёт к чтению с начала текста.
- Минимальные, проверяемые изменения в существующих модулях без смены авторского API `defineGame()`, если не выявится жёсткая необходимость.

**Non-Goals:**

- Полный аудит WCAG 2.2 или сертификация соответствия.
- Перевод всего UI на единый headless-компонентный фреймворк.
- Изменение визуального дизайна темы/шрифта для зрячих (кроме нейтральных обёрток без эффекта).

## Decisions

1. **Исключение «Назад»/«Вперёд» из пропуска Tab при «неактивности»**  
   Заменить или дополнить `disabled` паттерном, при котором кнопки остаются в порядке фокуса: например `aria-disabled="true"` + обработчик клика no-op + визуальный стиль «disabled» (как сейчас).  
   *Альтернатива*: оставить `disabled` и вставить невидимые фокусируемые «заглушки» — отклонено как хрупкое и плохое для UX.

2. **Удержание фокуса в модалках**  
   При открытии «Сохранения» или «Настройки» помечать основной корень страницы (всё вне панели диалога) как недоступный для фокуса и вспомогательных технологий через **`inert`** на обёртке основного приложения **или** на корневом контейнере вне диалога, плюс возврат фокуса на триггер при закрытии.  
   Если `inert` недоступен в целевых браузерах сборки — запасной вариант: ручной focus trap (keydown Tab на первом/последнем элементе) + `aria-hidden="true"` на фоне (осторожно с фокусом внутри скрытого).  
   *Решение*: предпочесть `inert` там, где политика проекта позволяет (современные evergreen); иначе задокументировать fallback в задачах.

3. **Озвучивание слота**  
   Связать `ifk-slot-num` и `ifk-slot-label` через **`role="group"`** с `aria-labelledby` (id на обоих span) или одним вычисляемым `aria-label` на группе карточки — чтобы при фокусе на первой кнопке действий чтец получил контекст слота. Избегать дублирования: если используется `aria-label` на группе, вложенные текстовые узлы не должны конфликтовать с именем.

4. **Тема и шрифт «вне чтеца»**  
   Обернуть секции в контейнер с **`aria-hidden="true"`**; убедиться, что внутри нет сфокусированного элемента при открытии модалки (фокус уходит в модалку первым действием open). Интерактивы темы/шрифта остаются для мыши/зрячих.

5. **Язык как «Музыка»/«Звуки»**  
   Ввести **`fieldset` + `legend`** для группы языков (или `role="group"` + видимый заголовок с `id` и `aria-labelledby`), чтобы структура совпадала по смыслу с подписанными ползунками.

6. **Фокус после сцены**  
   Пустой **`#ifk-scene-focus-anchor`** с `tabindex="-1"` в `#content` непосредственно перед `#scene-content`: в Tab не попадает, но принимает программный фокус после смены `sceneKey`, чтобы стрелка «вниз» у скринридера шла к началу текста сцены, не озвучивая весь `#controls`.

## Risks / Trade-offs

- **`inert` и полифиллы** — в старых движках WebView может не быть; нужен fallback → дублирование логики или условная ветка.  
  *Митигация*: feature-detect `HTMLElement.prototype.inert` или версия Tauri/WebView из `docs/building.md`.

- **`aria-hidden` на секции с фокусом** — риск «застрявшего» невидимого фокуса.  
  *Митигация*: при открытии модалки явно `focus()` на первый элемент диалога; при закрытии — снять `inert`/`aria-hidden` до возврата фокуса.

- **Якорь вне Tab** — при ошибочном включении в порядок Tab или дублировании id поведение ломается.  
  *Митигация*: только `tabindex="-1"`, один id, вставка из `engine` при старте.

## Migration Plan

- Изменения только в клиентском бандле; миграции данных и API не требуются.
- Откат: revert коммита(ов) модуля модалок и `scenes.ts` / разметки.

## Open Questions

- Нужно ли переносить фокус на якорь при первом запуске (стартовая сцена) — по умолчанию **да**, для единообразия, если не мешает UX тестов.
