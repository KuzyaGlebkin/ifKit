## 1. Тематические полосы прокрутки (CSS)

- [x] 1.1 В `src/style.css` ввести токены `--ifk-scrollbar-track` и `--ifk-scrollbar-thumb` (или эквивалент) в `:root`, `[data-theme="dark"]` и при необходимости в `@media (prefers-color-scheme: dark)` для согласованности с существующим каскадом темы; привязать к существующим `--ifk-color-*`.
- [x] 1.2 Добавить `scrollbar-color` и `scrollbar-width: thin` для корня/наследуемой области, плюс правила `::-webkit-scrollbar` / `::-webkit-scrollbar-thumb` / `::-webkit-scrollbar-track` с теми же переменными (селекторы по design: минимально достаточные, `html` и при необходимости контейнеры модалок).
- [x] 1.3 Проверить визуально в Chrome и в Firefox: светлая, тёмная, системная тема — полоса не «чужой» белой при тёмной теме.

## 2. Модалка настроек: внутренняя прокрутка

- [x] 2.1 Задать для `.ifk-modal-dialog` (или специфично для настроек, если отделите класс) `max-height` относительно viewport с запасом под padding backdrop, с fallback `100vh` при необходимости; сохранить flex-колонку.
- [x] 2.2 Сделать прокручиваемой область тела: для `.ifk-modal-body` в контексте настроек — `overflow-y: auto`, `min-height: 0`, `flex: 1 1 auto` (или эквивалент), чтобы шапка с `#ifk-settings-close` не уезжала; при необходимости уточнить разметку в `src/ifKit/settings-modal.ts` (обёртка), не ломая фокус-ловушку.
- [x] 2.3 Ручная проверка: ползунок шрифта 120–140% на невысоком окне — кнопка закрытия кликабельна, контент в теле прокручивается, полоса прокрутки соответствует теме.
