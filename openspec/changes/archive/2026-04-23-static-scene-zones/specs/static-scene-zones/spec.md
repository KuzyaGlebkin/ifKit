## ADDED Requirements

### Requirement: StaticContext — зонированный контекст для static-функции
Движок SHALL передавать в функцию `static` расширенный контекст `StaticContext<S, K>`, который включает все поля `SceneContext<S, K>` (`act`, `goto`, `local`) плюс три zone-функции: `before(cb)`, `after(cb)`, `slot(id, cb)`. Вызов `before(cb)` выполняет `cb` и весь HTML, записанный внутри, окажется перед содержимым сцены. Вызов `after(cb)` — после содержимого сцены. Вызов `slot(id, cb)` — в месте, где сцена объявила `Slot(id)`.

#### Scenario: static пишет контент перед сценой
- **WHEN** автор вызывает `before(() => { H2('Персонаж') })` внутри `static`
- **THEN** `<h2>Персонаж</h2>` рендерится в `#scene-content` до любого HTML, произведённого сценой

#### Scenario: static пишет контент после сцены
- **WHEN** автор вызывает `after(() => { P('Подвал') })` внутри `static`
- **THEN** `<p>Подвал</p>` рендерится в `#scene-content` после всего HTML сцены

#### Scenario: Множественные вызовы before конкатенируются
- **WHEN** автор вызывает `before(cb1)` и затем `before(cb2)` внутри `static`
- **THEN** в зоне before рендерится сначала результат `cb1`, затем `cb2`

#### Scenario: static вызывает act — кнопка появляется в zone действий
- **WHEN** автор вызывает `act('Инвентарь', cb)` внутри `static`
- **THEN** кнопка «Инвентарь» появляется в `#scene-acts` при каждой сцене

### Requirement: Тег-функция Slot — объявление именованного места в сцене
Движок SHALL экспортировать императивную тег-функцию `Slot(id: string): void`, которую автор вызывает внутри функции сцены. Вызов вставляет в поток HTML сцены маркер, который при compose заменяется контентом, записанным в `slot(id, cb)` функции `static`. `Slot` доступен в том же пространстве импортов, что и `H1`, `P`, и т.д.

#### Scenario: Сцена объявляет Slot, static заполняет его
- **WHEN** сцена вызывает `Slot('stats')` между `H1(...)` и `P(...)`, а static вызывает `slot('stats', () => { P('HP: 10') })`
- **THEN** в `#scene-content` рендерится: H1 → `<p>HP: 10</p>` → P

#### Scenario: Slot объявлен в сцене, но static его не заполняет
- **WHEN** сцена вызывает `Slot('foo')`, но static не вызывает `slot('foo', cb)`
- **THEN** маркер исчезает из итогового HTML (не оставляет пустого элемента), ошибки нет

#### Scenario: static заполняет слот, которого нет в сцене
- **WHEN** static вызывает `slot('bar', cb)`, но сцена не вызывает `Slot('bar')`
- **THEN** контент cb нигде не рендерится, ошибки нет

#### Scenario: Одна сцена объявляет несколько слотов
- **WHEN** сцена вызывает `Slot('a')` и `Slot('b')`, static заполняет оба
- **THEN** каждый слот заменяется своим контентом в позиции, где был объявлен

### Requirement: Отсутствие поля static сохраняет прежнее поведение
Если автор не передаёт поле `static` в `defineGame()`, движок SHALL работать без зонированного рендеринга — `#scene-content` содержит только HTML сцены, зоны before/after/slots не применяются.

#### Scenario: defineGame без static
- **WHEN** автор не передаёт поле `static`
- **THEN** игра запускается, `#scene-content` содержит только контент сцены
