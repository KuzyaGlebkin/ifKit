## Context

Кнопки `#ifk-settings-close` и `#ifk-saves-close` — иконка «✕» без текстовой подписи. Для AT имя элемента остаётся неоднозначным. В `settings-modal` уже есть механизм `data-ifk-ui-aria` и `refreshSettingsModalI18n()` для локализуемых `aria-label`.

## Goals / Non-Goals

**Goals:**

- Явное доступное имя для обеих кнопок закрытия: действие «закрыть» и контекст (настройки / сохранения).
- Строки через `u()` и ключи в `UI` / `builtin-ui`, обновление при смене языка UI.

**Non-Goals:**

- Менять видимый текст кнопки (остаётся «✕»).
- Менять порядок фокуса или логику закрытия.

## Decisions

1. **Имена ключей** — отдельные ключи, например `ui.settingsClose` и `ui.savesClose`, со значениями вроду «Закрыть настройки» / «Закрыть сохранения» (и EN-эквиваленты), чтобы не смешивать с заголовком модалки.
2. **Settings modal** — атрибут `data-ifk-ui-aria` на `#ifk-settings-close`, чтобы существующий проход в `refreshSettingsModalI18n()` выставлял `aria-label`.
3. **Saves modal** — тот же атрибут на `#ifk-saves-close`; в `refreshSavesModalChrome()` добавить обновление `aria-label` по `data-ifk-ui-aria` (короткий цикл по образцу настроек), т.к. отдельного универсального хелпера сейчас нет.

## Risks / Trade-offs

- [Дублирование 3–4 строк обновления aria в saves] → приемлемо; при росте числа таких элементов можно вынести общую функцию в отдельную задачу.
