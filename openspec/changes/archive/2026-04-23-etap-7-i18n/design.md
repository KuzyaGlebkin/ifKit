## Context

ifKit — single-file HTML движок для интерактивной фантастики. Весь код бандлится через `vite-plugin-singlefile`, поэтому внешние JSON-файлы локалей недоступны в рантайме — все строки обязаны быть инлайн. Автор пишет сцены на родном языке; переводы объявляются рядом в `defineGame()`. Движок уже имеет систему настроек (`Settings`, `localStorage`, модальное окно) — язык будет обычным полем настроек.

## Goals / Non-Goals

**Goals:**
- Автор пишет строки на родном языке прямо в коде; переводы — отдельно
- `` t`Строка` `` работает без конфигурации (возвращает оригинал если переводов нет)
- Язык сохраняется в `localStorage` вместе с остальными настройками
- Переключатель языка в UI настроек появляется автоматически когда `locales` переданы

**Non-Goals:**
- Плюрализация (1 монета / 2 монеты / 5 монет) — не в этом этапе
- RTL-языки — не в этом этапе
- Извлечение строк утилитой (аналог `xgettext`) — за рамками движка

## Decisions

### D1: Tagged template literal как синтаксис — не функция с ключом

**Выбрано**: `` t`Строка с ${var}.` ``  
**Альтернатива**: `t('some.key', { var })`

Rationale: автор не управляет ключами вручную — оригинальная строка и есть ключ. Это убирает целый класс ошибок (опечатки в ключах, расхождение ключа и содержимого). Fallback на оригинал работает естественно. Ближайший аналог — GNU gettext.

### D2: Ключ словаря — шаблон с позиционными плейсхолдерами

Для `` t`Найдено ${count} золота.` `` ключ строится как `"Найдено {0} золота."`.  
Переводчик пишет: `"Found {0} gold."`. Затем значения подставляются по индексу.

Rationale: именованные плейсхолдеры (`{count}`) требуют от автора дублировать имена переменных в строках словаря. Позиционные — проще и достаточны для большинства случаев IF-текстов.

### D3: `language` — поле `Settings`, хранится в `localStorage`

**Выбрано**: язык как обычная настройка.  
**Альтернатива**: отдельный ключ в `localStorage`.

Rationale: переиспользует уже существующую инфраструктуру (`loadSettings`, `applySettings`, `saveSettings`). Сброс настроек сбрасывает и язык к авторскому умолчанию — поведение согласованное.

`engineDefaults.language = ''` — пустая строка означает автоопределение из `navigator.language`. Автор может задать `settings: { language: 'ru' }` чтобы зафиксировать язык.

### D4: Инициализация `t` через module-level state в `i18n.ts`

`initI18n(locales, language)` вызывается из `defineGame()` один раз. Функция `t` читает module-level переменные (`_locales`, `_lang`). `setLanguage(lang)` меняет `_lang` и сохраняет в настройки.

Rationale: то же что у `audio.ts`, `snackbar.ts` — паттерн движка последовательный.

### D5: Переключатель языка в настройках — только если `locales` переданы

Секция «Язык» в `settings-modal.ts` рендерится только если в `GameConfig` есть непустой `locales`. Это не ломает существующие игры.

## Risks / Trade-offs

**[Risk] Хрупкость ключа**: изменение оригинальной строки в коде сцены разрывает связь с переводом в словаре — перевод молча перестаёт применяться, движок возвращает оригинал.  
→ Mitigation: документировать явно; в будущем можно добавить dev-warning в консоль при запуске в режиме разработки.

**[Risk] Коллизия ключей**: две разные строки с одинаковым текстом, но разным смыслом в разных контекстах получат один перевод.  
→ Mitigation: для IF-текстов это редкость; при необходимости автор может добавить неотображаемый разделитель (нулевой пробел) для уникализации.

**[Risk] Смена языка не перерисовывает текущую сцену автоматически**.  
→ Mitigation: смена языка происходит через `settings-modal` — после закрытия модального окна движок вызывает `rerender()` (тот же паттерн что при смене темы).
