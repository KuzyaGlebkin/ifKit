## ADDED Requirements

### Requirement: Ожидаемое заполнение UI для языка без встроенного `BUILTIN_UI`

Документация к движку SHALL явно указывать: для кода `code` присутствующего в `locales`, если **`BUILTIN_UI[code]`** не задан, слой `u` **до** авторского `locales[code].ui` **наследует** строки из встроенного fallback (текущее поведение: как минимум `BUILTIN_UI.en` в merge), поэтому chrome может оставаться на **английском**, пока автор **не** заполнит `ui` (например через `<code>.ui.json`). Требование **не** сужает движок к одному языку: относится к **любому** `code` вне покрытия `BUILTIN_UI`.

#### Scenario: Язык с полным `*.ui.json`

- **WHEN** для `code` нет `BUILTIN_UI[code]`, но `locales[code].ui` содержит **непустые** значения для всех `UiKey`, используемых текущим UI
- **THEN** `u` **не** откатывается к английскому из встроенного слоя **для** этих ключей

#### Scenario: Частично заполненный `ui`

- **WHEN** для ключа `UiKey` в `locales[code].ui` значение пропущено, а `BUILTIN_UI[code]` отсутствует
- **THEN** `u` для этого ключа использует значение по правилам merge (в т.ч. из `BUILTIN_UI.en` для недостающих строк), **без** отдельного нового ветвления в движке, если change не вводит иного алгоритма
