## ADDED Requirements

### Requirement: Источник стабильных ключей UI для `u()`

Проект SHALL трактовать перечисление `UiKey` (экспорт `UI` / тип `UiKey` из `src/ifKit/ui-keys.ts`) как **канонический** список идентификаторов системных строк для `u()`. Документация для автора SHALL указывать на этот модуль (или сгенерированный на его основе перечень) как на ответ на вопрос «какие бывают ключи navbar и модалок».

#### Scenario: Ключ панели инструментов адресуется стабильно

- **WHEN** в коде движка используется `u(UI.toolbarSettings)` (или эквивалентная строка ключа, совпадающая с `UiKey`)
- **THEN** семантика строки определяется сопоставлением с таблицей `UiKey`, а **не** с произвольной игровой фразой из `t`

### Requirement: Слияние встроенного и авторского UI

Документация SHALL объяснять, что **для каждого** кода языка в `locales` движок **сначала** наполняет слой `u` из встроенного `BUILTIN_UI` (с fallback на `en` при отсутствии **языка** в таблице встроенных слоёв), **затем** перекрывает значениями из `locales[code].ui`, если `LocaleBundle` с полем `ui` передан. Для языка **без** `BUILTIN_UI[code]` встроенный слой SHALL совпадать с `BUILTIN_UI.en` (или движковым эквивалентом) до внесения автора; заполнение `ui` **убирает** «английский» chrome, если все нужные значения заданы.

#### Scenario: Автор переписывает одну кнопку

- **WHEN** для языка `de` в `defineGame` задан `de: { game: {…}, ui: { "ui.settingsTitle": "…" } }` (или тот же ключ, что в `UI.settingsTitle`)
- **THEN** в интерфейсе на этом языке для заголовка настроек используется **авторское** значение, даже если в `BUILTIN_UI` есть встроенная строка для другого кода

### Requirement: Парные файлы `*.game.json` и `*.ui.json`

Шаблон и документация SHALL закреплять соглашение: для каждого кода `code`, под которым автор хранит `src/locales/<code>.game.json`, **рядом** в том же каталоге SHALL существовать `src/locales/<code>.ui.json` с картой «ключ `UiKey` — строка» для передачи в `locales[code].ui`, если автор ожидает локализованный chrome для этого кода. Исключения (только `game` в JSON, без `ui`) SHALL быть **осознанными** и задокументированы как сценарий с fallback на встроенный `en` для `u`.

#### Scenario: Переводчик открывает одну папку

- **WHEN** переводчик ищет все строки для кода `pl`
- **THEN** в `src/locales/` рядом лежат `pl.game.json` и `pl.ui.json` с непересекающимся смысловым назначением (игра vs chrome)

#### Scenario: Новый язык

- **WHEN** автор добавляет новый `xx.game.json`
- **THEN** в процессе/доке предусмотрено добавление `xx.ui.json` (вручную или через `dev-tooling`), иначе **известен** риск английского UI при локализованной игре
