## Why

В `src/locales/` уже принят шаблон **`*.game.json`** для игровых строк (`t`) и скрипт `i18n:extract` синхронизирует ключи с кодом. Для **системного** UI (`u`, navbar, модалки) действуют стабильные ключи `UiKey` и слой `BUILTIN_UI`, но в движке **есть встроенные переводы только для ограниченного набора языков** (например `en`, `ru`). Для **любого другого** кода языка, объявленного в `locales`, отсутствие записи в `BUILTIN_UI` означает неизбежный **fallback на английский** в chrome, пока автор не задаст `locales[code].ui`. Сейчас нет **симметричного** с игрой места хранения: неочевидно, что рядом с `game` нужен второй JSON, и не с чего копировать шаблон перевода UI.

## What Changes

- Ввести **соглашение о парных файлах**: для каждого `<code>.game.json` в шаблоне — **`<code>.ui.json`** с парами «ключ `UiKey` — строка» для слоя `ui` в `LocaleBundle` (импорты в `game.ts`: `game` + `ui`).
- Зафиксировать **источник правды по ключам**: `ui-keys` / `UI.*`; merge с `BUILTIN_UI` **как сейчас** (встроенное, затем автор).
- Добавить в **dev-tooling** генерацию/синхронизацию `*.ui.json` по списку `UiKey` (аналогично тому, как `i18n:extract` ведёт `*.game.json`): новые ключи — в файл, без машинного перевода; не ломать существующий extract для `t`.
- Обновить **документацию**: отличие `t` / `u`, пара файлов на язык, поведение для языка **без** `BUILTIN_UI[code]`.

## Capabilities

### New Capabilities

- `author-ui-strings`: Контракт пар `*.game.json` + `*.ui.json`, ключи `UiKey`, merge с `BUILTIN_UI`; явное различие игрового и chrome-слоёв.

### Modified Capabilities

- `define-game-api` / `project-scaffold`: Шаблон `defineGame` с импортом **двух** JSON на код языка при необходимости полного перевода chrome.
- `dev-tooling`: Скрипт(ы) или расширение `i18n:extract` для актуализации `*.ui.json` относительно `UiKey`.
- `i18n`: Уточнение ожидаемого полноты/источника UI для языков **без** встроенного слоя в `BUILTIN_UI` (автор несёт ответственность через `*.ui.json`, fallback на `en` только для **пропущенных** ключей, как в текущем merge).

## Impact

- `src/game.ts`, `src/locales/*`, `package.json` / `scripts/`, `docs/`.
- `src/ifKit` — **без** обязательного раздувания `BUILTIN_UI` под каждый новый язык; дополнительные встроенные языки в движке — опциональны и не заменяют пары JSON в шаблоне.
