## Why

Игра не может хранить предпочтения пользователя (тема, размер шрифта, громкость) между сессиями, и нет способа перенести прогресс между HTML-версией и Tauri-приложением. Этап 4 вводит систему настроек с персистентностью и возможностью ручного экспорта/импорта.

## What Changes

- Новый инфраструктурный модуль `storage.ts` — единая точка доступа к хранилищу для всего движка (LocalStorage / Tauri-заглушка / ручной экспорт-импорт)
- Новый модуль `settings.ts` — схема настроек v1, загрузка, сохранение, применение к DOM
- Расширение `GameConfig` — необязательное поле `settings` для авторских умолчаний
- UI: модальное окно настроек с немедленным применением изменений
- Кнопки «Экспорт» / «Импорт» (единый JSON-файл, namespace `ifkit:*`) и «Сбросить настройки» — внутри модалки

## Capabilities

### New Capabilities

- `storage-layer`: Абстракция хранилища с LocalStorageAdapter, TauriAdapter (заглушка) и функциями exportToFile / importFromFile
- `user-settings`: Настройки пользователя (тема, размер шрифта, заглушки громкости), авторские умолчания через GameConfig, применение к DOM
- `settings-modal`: Модальное окно настроек с элементами управления, кнопками сброса, экспорта и импорта

### Modified Capabilities

- `game-config`: Добавляется необязательное поле `settings: Partial<Settings>` в `GameConfig`

## Impact

- `src/ifKit/storage.ts` — новый файл
- `src/ifKit/settings.ts` — реализация модуля
- `src/ifKit/engine.ts` — чтение `config.settings`, инициализация настроек и модалки
- `src/ifKit/index.ts` — экспорт публичного API настроек
- `src/style.css` — стили модального окна
- `src/index.html` — без изменений (кнопка `#btn-settings` уже есть)
