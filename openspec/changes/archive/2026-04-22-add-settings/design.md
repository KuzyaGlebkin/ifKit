## Context

Движок не имеет механизма хранения пользовательских данных между сессиями. CSS-переменные для тем (`data-theme`, `--ifk-font-size-base`) и кнопка `#btn-settings` уже присутствуют в кодовой базе, но логика за ними отсутствует. Этап 4 вводит этот слой.

Этап 5 (сохранения) и Этап 8 (Tauri) будут использовать тот же механизм хранения, поэтому storage-абстракция должна быть спроектирована с учётом этих будущих потребителей.

## Goals / Non-Goals

**Goals:**
- Единый абстрактный Storage Layer для всего движка
- Настройки пользователя: тема, размер шрифта, заглушки громкости
- Авторские умолчания через `GameConfig.settings`
- Модальное окно настроек с немедленным применением изменений
- Ручной экспорт/импорт единым JSON-файлом
- Кнопка «Сбросить настройки» (возврат к авторским умолчаниям)

**Non-Goals:**
- Настоящая реализация Tauri-адаптера (заглушка)
- Интеграция громкости с аудиодвижком (Этап 6)
- Реализация сохранений игры (Этап 5)
- Синхронизация с облаком / Steam Cloud

## Decisions

### 1. Storage Layer как синглтон с авто-детектом

Единственный `storage` объект инициализируется при первом обращении. Авто-детект: `'__TAURI__' in window` → `TauriAdapter`, иначе `LocalStorageAdapter`.

**Альтернатива:** явная конфигурация в `GameConfig({ storage: 'local' | 'tauri' })`.
**Отклонено:** лишний бойлерплейт для автора; в вероятных сценариях авто-детект всегда даёт верный результат.

### 2. Namespace `ifkit:` в ключах хранилища

Все ключи движка используют префикс `ifkit:` (`ifkit:settings`, `ifkit:saves`). Пространство имён исключает коллизии с кодом автора.

### 3. Единый JSON-файл для экспорта

Экспорт собирает все ключи `ifkit:*` в один файл с версией схемы. Импорт восстанавливает все ключи.

**Альтернатива:** раздельные файлы для настроек и сохранений.
**Отклонено:** целевой сценарий — «перенести всё» между HTML и Tauri; один файл удобнее.

### 4. Сброс к авторским умолчаниям, не к движковым

`resetSettings()` восстанавливает значения из `GameConfig.settings` (слитые с движковыми defaults), а не жёсткие движковые defaults.

**Мотивация:** разные игры имеют разные «правильные» настройки; автор задумал определённый опыт, сброс должен уважать этот замысел.

### 5. Немедленное применение изменений в модалке

Изменения применяются к DOM при каждом взаимодействии с элементом управления и одновременно сохраняются в хранилище. Кнопки «Сохранить» / «Отменить» отсутствуют.

**Альтернатива:** применение только при закрытии / по кнопке «Сохранить».
**Отклонено:** немедленный отклик даёт более предсказуемый UX; для IF-игры накладные расходы на запись в localStorage незначительны.

### 6. Заглушки громкости

Поля `musicVolume` и `soundVolume` хранятся и отображаются в UI, но не подключены к аудиодвижку. При инициализации Этапа 6 аудио само читает эти значения из `settings`.

## Risks / Trade-offs

- **TauriAdapter как заглушка** → при сборке Tauri настройки пишутся в localStorage, что на некоторых платформах может не работать. Митигация: в Этапе 8 TauriAdapter заменяется настоящей реализацией; заглушка явно помечена комментарием `// TODO Этап 8`.
- **Нет кнопки «Отменить» в модалке** → пользователь не может "попробовать" настройку и откатить. Митигация: кнопка «Сбросить настройки» доступна всегда.
- **`version: 1` в формате экспорта без механизма миграции** → если схема изменится в v2, импорт старого файла может дать неверный результат. Митигация: при несовпадении версии выводить предупреждение и предлагать сброс.
