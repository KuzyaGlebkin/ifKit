## Context

Сейчас `defineGame` после инициализации сразу вызывает игровой цикл с первой сценой (`runGameLoop(firstKey)`), навбар всегда виден, музыка может стартовать через `PlayMusic` в первой сцене до любого осознанного действия пользователя. Модалка сохранений единообразно предлагает сохранение и перезапись. Логотип в `#controls` по спецификации не должен быть ловушкой фокуса и в основном декоративен.

## Goals / Non-Goals

**Goals:**

- Показать полноэкранное стартовое меню без `#controls` при первом заходе в **сессию** (вкладка/окно), с центрированной колонкой: логотип, название, описание автора, три кнопки (Новая игра с Lucide `Plus`; Загрузить и Настройки — те же SVG/иконки, что у `#btn-saves` и `#btn-settings`).
- По клику на логотип в навбаре: записать текущее состояние в **авто-слот** (как принято в движке), остановить фоновую музыку, показать то же меню (навбар скрыть).
- Открыть модалку сохранений из меню «Загрузить» в режиме **только загрузка** (без UI и логики сохранения/перезаписи слотов).
- Открыть модалку настроек из меню с тем же поведением, что и `#btn-settings`.
- Дать автору способ задать **название игры** и **краткое описание** для меню (и локализуемость через существующие механизмы `t` / `locales` по решению реализации).
- После выхода из меню в игру сохранять текущие правила аудио (глобальное «не играть музыку в меню», не перетирать пользовательские настройки).

**Non-Goals:**

- Отдельный экран главного меню вне браузера/Tauri без изменения общей модели сохранений.
- Обязательный сброс прогресса при каждом заходе в меню по логотипу (по умолчанию — только автосохранение и переход в меню).
- Новый способ удалять сохранения Tauri через UI оболочки ОС — только упоминается как контекст (браузер vs десктоп).

## Decisions

1. **Состояние «стартовое меню активно»**  
   Ввести явное состояние движка (например `sessionUi: 'menu' | 'playing'`), персистентное только в памяти страницы: при первой загрузке — `'menu'`, после «Новая игра» или успешной «Загрузить» из слота — `'playing'`. Обновление страницы сбрасывает к показу меню заново (новая сессия).  
   **Альтернатива**: `sessionStorage` — отклонена для первого этапа, чтобы совпасть с формулировкой «первый запуск в сессии» без дополнительного persisted флага.

2. **Порядок инициализации**  
   Не запускать `runGameLoop` с первой сценой до выбора «Новая игра» или загрузки из слота; либо запускать цикл только после этого (предпочтительно не рендерить игровую сцену под меню, чтобы не вызывать `PlayMusic` раньше времени).  
   **Альтернатива**: рендер сцены с нулевой громкостью — хуже для контракта аудио и скринридера.

3. **Логотип в навбаре**  
   Сделать логотип (или его обёртку) интерактивной кнопкой/ссылкой с явным именем для AT (`aria-label` из UI-строк). По активации: вызвать существующий путь автосохранения в авто-слот, перевести аудио в состояние «меню» (см. п. 4), показать слой меню и скрыть `#controls`/основную сцену визуально по правилам спеки.

4. **Музыка в меню**  
   При входе в стартовое меню остановить/заглушить воспроизведение музыки (через существующий intent/clear/stop музыкального источника), не изменять сохранённые пользователем `musicMuted`/`musicVolume` в настройках. При возврате в игру применять обычные правила сцены и настроек.  
   **Альтернатива**: принудительно выставлять mute в настройках — отклонена.

5. **Режим модалки сохранений «только загрузка»**  
   Параметр открытия (например `openSavesModal({ mode: 'load-only' })` или эквивалент), при котором скрываются кнопки «Сохранить», режим перезаписи и связанный UI; сохранены сценарии загрузки, удаление при необходимости можно запретить в этом режиме или оставить — зафиксировать в спеке как «только действия загрузки» (минимум: нет сохранения/перезаписи).

6. **Метаданные игры для меню**  
   Расширить `defineGame` опциональным полем (например `meta: { title: string; description?: string }` или строки, проходящие через `t` при необходимости). Пустые значения — недопустимы для заголовка в шаблоне или fallback на имя проекта документирован отдельно в спеке.

7. **Иконки**  
   Новая игра — `Plus` из `lucide` (или уже используемый в проекте способ svg-иконок). Загрузить/Настройки — повторное использование разметки/спрайтов тех же иконок, что у `#btn-saves` и `#btn-settings`, чтобы визуально совпасть с навбаром.

## Risks / Trade-offs

- **[Risk]** Два источника открытия модалки сохранений (навбар и меню) и разный режим → **Mitigation**: один API открытия с обязательной семантикой режима; тесты на отсутствие кнопок сохранения в `load-only`.  
- **[Risk]** Изменение порядка старта заденет авторов, которые ожидали немедленный `runGameLoop` побочные эффекты → **Mitigation**: явное описание в `define-game-api` и миграция шаблонной игры.  
- **[Risk]** Фокус и порядок Tab на полноэкранном меню и при возврате в игру → **Mitigation**: следовать паттерну модалок/landmark, отдельные сценарии в спеках `session-main-menu` и `project-scaffold`.

## Migration Plan

- Обновить образцовую игру (`game.ts`, локали при необходимости) с заполнением `meta` и проверкой первого экрана.  
- После выпуска: авторы добавляют `meta`/строки; без полей — задокументированный fallback.  
- Откат: фичефлаг или возврат к немедленному `runGameLoop` не требуется, если сохранять обратную совместимость через опциональные поля.

## Open Questions

- Нужен ли в режиме «только загрузка» **удаление** слотов (может облегчить «новый старт» без файловой системы) — оставить на усмотрение первой реализации с минимальным набором «только загрузка без записи».
