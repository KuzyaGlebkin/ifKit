## Context

В `Settings` и аудиодвижке по-прежнему используются флаги `musicMuted` / `soundMuted`: `true` означает заглушённую ветку. Модалка настроек привязывала чекбокс напрямую к этим полям (`checked === muted`), а подписи/aria опирались на «mute», из-за чего ожидаемая семантика «галочка = включено» не соблюдалась.

## Goals / Non-Goals

**Goals:**

- В UI: **включённый** чекбокс = ветка **слышна** (хранилище: `*Muted: false`).
- Сохранить формат JSON и поля `musicMuted` / `soundMuted` без миграции и без смены контракта `audio-engine` / `setMusicMuted` / `setSoundMuted`.
- Обновить строки i18n и aria так, чтобы они соответствовали положительной семантике.

**Non-Goals:**

- Переименование полей в `Settings` (например `musicEnabled` вместо `musicMuted`).
- Изменение формул эффективного gain в `audio.ts`.

## Decisions

1. **Инверсия только на границе UI**  
   При открытии модалки: `checkbox.checked = !settings.musicMuted` (и аналогично для звуков). При `change`: `commit({ musicMuted: !target.checked })`. Аудио-API по-прежнему получает булевы `muted` как сейчас.

2. **Ключи i18n**  
   Предпочтительно переименовать ключи в `ui-keys` с `*Mute` на `*Enabled` (или `*On`) и обновить `builtin-ui`/хром-строки, чтобы авторы не путали семантику. Если переименование нежелательно в одном PR — допустимо оставить стабильные идентификаторы и заменить только тексты fallback (меньше правок в типах, но имена ключей остаются вводящими в заблуждение).

3. **Доступность**  
   `aria-label` / связанный ключ для control SHALL отражать «включить музыку» / «включить звуки» (или эквивалент), согласованно с видимой подписью секции.

## Risks / Trade-offs

- [Авторы переопределяли старые строки «mute» в `locales`] → Документировать в proposal как **BREAKING** для кастомных словарей; при реализации обновить список ключей в коммите/заметке.
- [Дублирование логики `!`] → Свести к двум местам в `settings-modal` (init + handler), без расхождений.

## Migration Plan

- Не требуется миграция хранилища: существующие сохранения с `musicMuted: true` отобразятся как **снятый** чекбокс «включения» — соответствует прежнему «mute включён».

## Open Questions

- Нужны ли новые стабильные ключи `ui.settingsMusicEnabled` вместо `ui.settingsMusicMute` (команда может выбрать при реализации по объёму правок).
