## Why

Фоновая музыка и речь скринридера (NVDA, JAWS, Narrator и др.) смешиваются в общей громкости Windows; при типичных настройках громкости игры голос внешнего AT становится плохо различим. В веб-контексте нельзя надёжно определить момент речи внешнего скринридера, поэтому нужна явная настройка, снижающая уровень фоновой музыки в приложении, чтобы пользователь мог слышать синтез речи.

## What Changes

- Новый флаг в настройках (персистентный): **снижение громкости фоновой музыки для совместимости со скринридером** — при включении эффективная громкость музыки умножается на фиксированный коэффициент (значение задаётся в реализации, см. design), при этом сохранённые `musicVolume` и поведение mute не перезаписываются.
- Модалка настроек: переключатель с поясняющей подписью/подсказкой (i18n), размещение в секции аудио рядом с управлением музыкой.
- Аудиодвижок: учёт флага при расчёте `_masterMusicGain` вместе с номиналом и mute; применение при смене настроек без перезапуска.
- Старые сохранения без поля: трактовать как «выключено» (музыка как сейчас).

### Modified Capabilities

- `user-settings`: новое поле в `Settings`, `engineDefaults`, миграция/загрузка и применение при старте.
- `audio-engine`: эффективная громкость музыки с учётом флага «тихая музыка для AT»; публичный API синхронизации с движком настроек (как для остальных аудиополей).
- `settings-modal`: UI переключателя, порядок фокуса и доступность (связка label / описание).
- `i18n`: ключи и встроенные строки для подписи и краткой подсказки переключателя.

## Impact

- `src/ifKit/settings.ts` (или модуль схемы), `settings-modal.ts`, `audio.ts`, связка в `engine.ts` при инициализации/обновлении настроек.
- `ui-keys.ts`, `builtin-ui.ts` при добавлении ключей.
