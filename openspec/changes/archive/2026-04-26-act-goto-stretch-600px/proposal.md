## Why

На ширинах между 481px и 600px кнопки сцен (`act` и `goto`) остаются «инлайновой» ширины и в узкой колонке контента выглядят слишком узко и неудобно для касания. Тот же брейкпоинт **600px** уже используется для компактной панели `#controls`; выравнивание порога «растягивания» кнопок сцен с ним делает поведение предсказуемым на типичных мобильных ширинах и малых окнах Tauri.

## What Changes

- Перенос CSS-правил полной ширины (и выравнивания текста) для `.btn-act` и `.btn-goto` с порога **480px** на порог **ширина viewport ≤ 600px**.
- Сохранение остальных правил блока `@media (max-width: 480px)` для `#content` (отступы) без изменения поведения, если они не конфликтуют с новым порогом для кнопок.
- Обновление спецификации `ui-buttons`: зафиксировать новый брейкпоинт для полной ширины кнопок сцен; при необходимости уточнить сценарии про узкий экран.

## Capabilities

### New Capabilities

_(нет — поведение остаётся тем же по смыслу, меняется только порог ширины)_

### Modified Capabilities

- `ui-buttons`: добавить/уточнить требование о полной ширине `btn-act` / `btn-goto` при ширине viewport **не более 600px**; при необходимости скорректировать сценарии, где фигурирует 480px только в контексте переноса строк.

## Impact

- `src/style.css` — медиазапросы для сценовых кнопок и, при необходимости, группировка с существующим `@media (max-width: 600px)` для тулбара.
- `openspec/specs/ui-buttons/spec.md` — дельта после архивации изменения.
