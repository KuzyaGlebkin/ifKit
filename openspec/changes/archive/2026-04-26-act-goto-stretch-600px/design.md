## Context

В `src/style.css` полная ширина и центрирование текста для `.btn-act` и `.btn-goto` заданы внутри `@media (max-width: 480px)` вместе с уменьшенным padding у `#content`. Отдельный блок `@media (max-width: 600px)` уже скрывает подписи на `#controls` и в модалке сохранений. Пользователь хочет, чтобы сценовые кнопки переходили в режим «на всю ширину строки» при **ширине viewport ≤ 600px**, синхронно с компактным тулбаром.

## Goals / Non-Goals

**Goals:**

- При `max-width: 600px` (включительно): каждая `.btn-act` и `.btn-goto` занимает полную ширину своей flex-строки в `#scene-acts` / `#scene-gotos`, текст по центру (как сейчас при 480px).
- Правило padding для `#content` при `max-width: 480px` оставить без изменений.

**Non-Goals:**

- Менять брейкпоинт 480px для модалок и прочих компонентов.
- Менять `flex-wrap` или отступы зон `#scene-acts` / `#scene-gotos`, кроме переноса правил ширины кнопок.

## Decisions

1. **Порог 600px, формулировка «не более 600px»** — в CSS это `@media (max-width: 600px)`, что соответствует требованию «ширина меньше либо равна 600px».

2. **Где разместить правила** — добавить `.btn-act` / `.btn-goto` в уже существующий блок `@media (max-width: 600px)` у тулбара (конец файла, около строки 614), с кратким комментарием «scene buttons», чтобы не плодить второй медиаблок с тем же условием. Альтернатива — отдельный `@media (max-width: 600px)` сразу после стилей сценовых зон; оба варианта эквивалентны для браузера; предпочтение — один блок 600px для согласованности «компактная ширина».

3. **Удалить** дублирующие правила ширины кнопок из `@media (max-width: 480px)` (оставив там только `#content` padding).

## Risks / Trade-offs

- **[Risk]** На ширине 481–600px кнопки станут выше по занимаемой горизонтали → **Mitigation**: это запрошенное поведение; вертикальный скролл при многих кнопках остаётся за счёт `flex-wrap`.

- **[Risk]** Визуальная плотность на очень широких, но низких окнах не затрагивается (медиа по ширине, не по высоте) → **Non-goal** для этой задачи.

## Migration Plan

Не применимо: только фронтенд-CSS и спека; откат — вернуть правила в блок 480px.
