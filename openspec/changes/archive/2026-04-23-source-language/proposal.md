## Why

Если автор объявляет переводы только для одного языка (например `en`), движок не знает об оригинальном языке: переключатель языков не появляется в настройках, а при английской локали браузера игра стартует сразу на переводе. Автор лишён возможности предоставить игрокам выбор языка без обходных путей.

## What Changes

- В `GameConfig.defineGame()` добавляется необязательное поле `sourceLanguage: string` — явное указание кода языка оригинальных строк
- При инициализации движок автоматически включает `sourceLanguage` в список доступных языков и в словарь локалей (как пустой перевод — fallback на исходный текст)
- Переключатель языка в настройках корректно отображается, когда есть оригинал + хотя бы один перевод

### Modified Capabilities

- `define-game-api`: новое необязательное поле `sourceLanguage` в `GameConfig`
- `i18n`: `initI18n` получает `fullLocales` с явно добавленным `sourceLanguage: {}`, что корректно направляет автоопределение языка по браузеру

## Impact

- `src/ifKit/engine.ts` — единственный файл с логическими изменениями: построение `fullLocales` и `languages`
- `src/game.ts` (пример игры) — добавить `sourceLanguage: 'ru'`
- Без `sourceLanguage` поведение полностью идентично текущему (обратная совместимость)
