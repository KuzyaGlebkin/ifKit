## Context

- Web-сборка использует `vite-plugin-singlefile`: в `dist/` остаётся один `index.html`; внешние `href` на отдельные файлы рядом с `index.html` для `file://` ненадёжны.
- Tauri-фронт собирается в `dist-tauri/` с обычными ассетами; вкладка WebView2 читает `index.html` как мультифайловый бандл.
- Мастера графики лежат в `docs/if_fav.svg` (сейчас тёмная заливка `#343434` на прозрачном/светлом холсте) и `docs/logo.svg` (чёрная заливка). Для продакшена нужны версии, соответствующие ТЗ: favicon — **белый** глиф на **тёмном** фоне; логотип в UI — **акцент** темы.
- Нативные иконки Tauri уже перечислены в `tauri.conf.json` (`src-tauri/icons/*.png`, `.ico`, `.icns`).

## Goals / Non-Goals

**Goals:**

- Favicon во вкладке для `npm run build` и для Tauri webview (`build:tauri-web`), визуально согласованный с `if_fav` (белый на тёмном).
- Логотип в начале `#controls`, высота согласована с панелью, цвет следует за `var(--ifk-color-accent)` (включая смену темы и пресеты из настроек).
- Обновить бинарные иконки приложения Tauri из того же бренда, не ломая конфиг.

**Non-Goals:**

- Смена текста/формы логотипа или типографики игры.
- Отдельный «логотип для печати» или маркетинговые экспорты.
- Локализация графического слова «ifKit» (остаётся единым SVG).

## Decisions

1. **Favicon в single-file web**  
   **Решение:** хранить итоговый favicon как **data URL** (`data:image/svg+xml,...`) в `src/index.html` или как результат сборки, который попадает в единственный `dist/index.html` без отдельного файла.  
   **Альтернатива:** файл в `public/` — отклонена: при открытии только `dist/index.html` с диска относительный путь к favicon часто недоступен.

2. **Исходник favicon**  
   **Решение:** завести в репозитории **production SVG** (например `src/ifk-favicon.svg` или рядом с ассетами), полученный из `docs/if_fav.svg`: фон — тёмный (например `#1a1a1a` или согласованный с `--ifk-color-bg` тёмной темы), силуэт — `#ffffff`. Inkscape-метаданные SHALL быть убраны для компактности.  
   **Альтернатива:** править только docs-файл — отклонена, чтобы не ломать авторский экспорт в Inkscape.

3. **Логотип в панели**  
   **Решение:** **inline SVG** в разметке `#controls` (или один импортируемый фрагмент), заливки через `currentColor`, обёртка с `color: var(--ifk-color-accent)`. Пути взять из `docs/logo.svg` (упрощённый root без sodipodi).  
   **Альтернатива:** `<img src="...">` + CSS `filter` — хрупко для смены акцента.

4. **Tauri `icons/`**  
   **Решение:** перегенерировать стандартным инструментом (`npx @tauri-apps/cli icon …`) из **квадратного** мастера (PNG 1024×1024 или SVG, если CLI поддерживает), визуально совпадающего с favicon (белое на тёмном). Зафиксировать в `tasks.md` команду для повторяемости.  
   **Альтернатива:** ручная сборка всех размеров — только если CLI недоступен.

5. **Размещение в `#controls`**  
   **Решение:** логотип — **первый** визуальный элемент слева (до кнопок), декоративный для AT с осмысленным `aria-label` на ссылке/контейнере (`aria-hidden` на самом SVG при наличии подписи). Не должен красть фокус Tab у кнопок: использовать контейнер без `tabindex` или `tabindex="-1"` по согласованию с `ui-zones` (логотип не в цепочке Tab).

## Risks / Trade-offs

- **[Risk]** Размер data URL в head увеличивает HTML.  
  **→ Mitigation:** минифицировать SVG (один path, без метаданных).

- **[Risk]** Расхождение между data URL favicon и PNG для Tauri.  
  **→ Mitigation:** один визуальный макет; PNG экспорт из того же SVG/мастера.

- **[Risk]** Слишком мелкий wordmark на узкой ширине.  
  **→ Mitigation:** `max-height` в `rem`, при необходимости скрывать wordmark на том же breakpoint, что и icon-only кнопки (по согласованию с реализацией — зафиксировать в задачах при ревью).

## Migration Plan

- Нет данных пользователя. Деплой: обычная сборка; откат — вернуть предыдущие `icons/` и разметку из git.
