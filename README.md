# ifKit

Движок интерактивной литературы на TypeScript. Пишите игры с разветвлёнными историями, используя знакомый TypeScript — без собственных языков и GUI-редакторов.

Собирается в один самодостаточный HTML-файл: скачать и запустить в браузере без сервера.

---

## Минимальный пример

```ts
// game.ts
import { defineGame } from './src/ifKit'
import { scenes } from './scenes'
import { initialState } from './state'

defineGame({
  state: initialState,
  static: (state) => {
    P(`HP: ${state.hp}`)
  },
  scenes,
})
```

```ts
// state.ts
export const initialState = {
  hp: 10,
  gold: 0,
}

export type GameState = typeof initialState
```

```ts
// scenes.ts
import type { GameState } from './state'

export const scenes = {
  start: (_state: GameState) => {
    P('Ты в лесу.')
    goto('clearing')
  },
  clearing: (_state: GameState) => {
    P('На поляне тихо.')
  },
}
```

> `P`, `goto` и другие теговые функции появятся в Этапе 1 (рендеринг и сцены).

## Начало работы

### 1. Скачайте шаблон

Скачайте ZIP-архив репозитория и распакуйте в папку вашей игры.

### 2. Установите зависимости

```bash
npm install
```

### 3. Запустите режим разработки

```bash
npm run dev
```

Откройте `http://localhost:5173` в браузере. При сохранении файлов страница обновляется автоматически.

### 4. Соберите игру

```bash
npm run build
```

Файл `dist/<name>_<version>.html` (например `ifkit_0.1.0.html` для текущего `package.json`) — готовая игра. Отправьте его другу, разместите на хостинге или запустите в Tauri.

## Структура проекта

```
my-game/
├── index.html        ← оболочка (не трогайте без необходимости)
├── style.css         ← стили (можно и нужно менять)
├── game.ts           ← точка входа вашей игры
├── scenes.ts         ← сцены вашей игры
├── state.ts          ← тип и начальное состояние
├── assets/
│   ├── audio/        ← звуки и музыка
│   └── images/       ← изображения
└── src/
    └── ifKit/        ← движок (не трогайте без необходимости)
        └── index.ts  ← публичный API движка
```

### Файлы автора

| Файл        | Описание                                                                                                         |
| ----------- | ---------------------------------------------------------------------------------------------------------------- |
| `game.ts`   | Точка входа. Вызывает `defineGame()` с конфигурацией игры.                                                       |
| `scenes.ts` | Объект со сценами. Каждая сцена — функция, описывающая содержимое экрана.                                        |
| `state.ts`  | Начальное состояние (`initialState`) и тип `GameState`. Выносится отдельно, чтобы избежать циклических импортов. |

### `defineGame(config)`

| Поле     | Тип                               | Обязательное | Описание                                          |
| -------- | --------------------------------- | :----------: | ------------------------------------------------- |
| `state`  | `object`                          |      ✓       | Начальное состояние игры                          |
| `scenes` | `Record<string, (state) => void>` |      ✓       | Объект со сценами                                 |
| `static` | `(state) => void`                 |      —       | Постоянная зона (характеристики персонажа и т.п.) |

## Расширение движка

Движок находится в `src/ifKit/`. Каждый файл отвечает за одну фичу:

| Файл             | Описание                           |
| ---------------- | ---------------------------------- |
| `renderer.ts`    | Буфер рендеринга и теговые функции |
| `scenes.ts`      | Реестр и переходы между сценами    |
| `state.ts`       | Управление состоянием игры         |
| `saves.ts`       | Сохранения и история               |
| `audio.ts`       | Аудиодвижок                        |
| `i18n.ts`        | Интернационализация                |
| `layout.ts`      | Зоны интерфейса                    |
| `transitions.ts` | Анимации переходов                 |
| `snackbar.ts`    | Уведомления об изменениях          |
| `settings.ts`    | Настройки пользователя             |

Замените любой модуль — остальное продолжит работать.

## Команды

| Команда                    | Описание                                     |
| -------------------------- | -------------------------------------------- |
| `npm run dev`              | Запустить dev-сервер с HMR                   |
| `npm run build`            | Web-сборка: один `<name>_<version>.html` в `dist/` |
| `npm run lint`             | Проверить код ESLint                         |
| `npm run format`           | Форматировать код Prettier                   |
| `npm run tauri:dev`        | Нативное окно поверх dev-сервера             |
| `npm run tauri:build`      | Desktop-установщик (Windows / macOS / Linux) |
| `npm run tauri:android`    | Android APK/AAB                              |

## Сборка и публикация

Подробная инструкция по всем типам сборки, настройке окружения для Android и конфигурации Steam Auto-Cloud — в [`docs/building.md`](docs/building.md).

**Кратко:**
- **Web**: `npm run build` → `dist/<name>_<version>.html` — открывается без сервера, публикуется на itch.io / GitHub Pages
- **Desktop**: `npm run tauri:build` → нативный установщик для Windows, macOS, Linux
- **Android**: `npm run tauri:android` → APK (требует Android Studio и NDK)
